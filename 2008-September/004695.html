> > Hi everybody!!
> > I have installed the OpenIMS Core in a computer server and all run OK!
> > I have tried a call between Alice and Bob and all seems to work fine.
> > My next step is to install a Presence Server but I have a lot of doubts:
> > -I don't know if I have to install it in another machine using for
> > example OpenSIPs or if I can use some OpenIMS module in the same machine.
> the presence module from Open IMS is a very old version of the presence
> module from SER (iptel.org). I would go for a newer one. Then you can
> install it on the same machine but just don't choose ports 4060, 5060 or
> 6060 on the same IP address as the Open IMS Core. If you use it on
> another machine, please remember to reconfigure you Open IMS Core on a
> non local-loopback IP address, or else it won't be able to communicate
> with the presence server or anything outside the Open IMS system.
> 
> > -Also I do not know which are the steps to follow to install a
> > presence server that works with the OpenIMS system: which files I have
> > to modify, which ports are involved...??
> any should work. Then in the HSS configured for those users, there's a
> default trigger point that looks like for presence and you can use that.
> Change the default AS configuration to your particular SIP URI of your
> presence server.
> 
> Although the default trigger point works in a simple initial setup, it
> is a good idea to change the default trigger point such that it would
> trigger PUBLISH only when Session-Case==originating and SUBSCRIBE only
> when Session-Case==terminating or terminating-unregistered. This is to
> make user that multi-domain scenarios will also work.
> > -Which presence server do you recommend me??
> I don't know... any would do...
> 
> The Open IMS Playground @FOKUS contains one (
> http://www.fokus.fraunhofer.de/en/fokus_testbeds/open_ims_playground/components/presence_server/features/index.html
> ), with features like RLS or Parlay X interfaces, but it's not open source
