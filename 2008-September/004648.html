Hello,

We're facing a strange issue for our testbed. We've deployed 3 HQs with 
one S-CSCF and one P-CSCF for each. We've found the following issue:

- If the HSS database has 6 users pre-provisioned.
- If, from one HQ, a registered user tries to call to a provisioned user 
that is not registered --> the calling S-CSCF automatically chooses itself 
as the serving S-CSCF for the called user.
- That calling S-CSCF register the called user as "unregistered"
- The association between that S-CSCF<->IMSU is never deleted.

Is this behaviour normal?
How can we avoid this behaviour?

I attach you a file with the calling S-CSCF log. Users 5555555 & 1111111 
are auto unregistered.
