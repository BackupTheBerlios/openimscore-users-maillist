Having thought about this some more. If the array is a has been defined as a char type rather than unsigned char then you might be able to use a conversion modifier to fix this problem. Try replacing %d with %hhu. This should effectively force a conversion to unsigned char. And the values -64 and -88 should print as 192 and 168
