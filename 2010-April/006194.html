Hi All,

We are testing the functionality of an application server which acts as a
Proxy in between the Endpoints and the Open-IMS server.

Our set-up looks like this:
There are two IMPUs - krish at open-ims.test and kaushik at open-ims.test
There is one Extension 26013189 at 10.232.37.233 which is associated with the
IMPU krish at open-ims.test

When the Extension 26013189 at 10.232.37.233 registers with the application
server, krish at open-ims.test will be registered with the Open-IMS server.

When the Extension 26013189 at 10.232.37.233 makes a call to
kaushik at open-ims.test, the application server changes the From-header to
krish at open-ims.test and sends the INVITE to open-IMS server. And when
kaushik at open-ims.test sends the responses, the application server changes
the From-header to 26013189 at 10.232.37.233 and sends it to the extension. The
application server takes care of adding Record-Route headers to ensure that
the requests/responses go through it.

In this scenario we observed that the ACK sent by the Extension
26013189 at 10.232.37.233 is getting dropped at the Open-IMS server. We also
observed that the Route headers are populated properly based on the
Record-Route hdrs. But we are unable to understand why the Open-IMS server
is dropping this ACK.

Any inputs on  this issue would be helpful.

I have attached the ethereal traces of this scenario for your reference. I
have also attached the ethereal traces of Successful call scenario between
the endpoints directly registered with the Open-IMS server without our
application server.

Thanks in advance.

For your reference:
Open-IMS server is on 10.232.15.41:4060
Application server is running on 10.232.37.233:6060
The Endpoints are on 10.232.37.233

Regards,
N.V.S.Kaushik
