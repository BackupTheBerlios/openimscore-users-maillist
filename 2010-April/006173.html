Hi all,

I have configured open-ims i-cscf and s-cscf on the same machine. When 
the originating s-cscf sends INVITE to the terminating s-cscf, it 
responds with ICMP message (port unreachable). while netstat gives -
#netstat -an | grep 6060
tcp        0      0 192.168.169.19:6060         
0.0.0.0:*                   LISTEN
udp        0      0 192.168.169.19:6060         0.0.0.0:*

Registration is successful but INVITE session is not through.
If I use dig I get the correct result -
#dig @192.168.169.19 scscf.open-ims.test
; <<>> DiG 9.4.1-P1 <<>> @192.168.169.19 scscf.open-ims.test
; (1 server found)
;; global options:  printcmd
;; Got answer:
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 15100
;; flags: qr aa rd ra; QUERY: 1, ANSWER: 1, AUTHORITY: 1, ADDITIONAL: 1
;; QUESTION SECTION:
;scscf.open-ims.test.           IN      A
;; ANSWER SECTION:
scscf.open-ims.test.    86400   IN      A       192.168.169.19
;; AUTHORITY SECTION:
open-ims.test.          86400   IN      NS      ns.open-ims.test.
;; ADDITIONAL SECTION:
ns.open-ims.test.       86400   IN      A       127.0.0.1
;; Query time: 1 msec
;; SERVER: 192.168.169.19#53(192.168.169.19)
;; WHEN: Tue Apr  6 05:11:52 2010
;; MSG SIZE  rcvd: 86

Could anybody help me what could be the error?
pcap trace, open-ims.dnszone file, /etc/hosts, /etc/named.conf, 
scscf.cfg, scscf.xml and /etc/resolv.conf are attached.
