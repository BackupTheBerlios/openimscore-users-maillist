The HSS has profile part indicators, you should be able to apply different actions for online (i.e., registered) and offline (i.e., not registered) users.

I don’t have the configuration front-end accessible just now, but I think to remember it was under Service - iFC. Not sure though, haven't tested it yet.

I guess if you set it for unregistered, then the trigger will only apply if the user is not registered. If I understand you correctly, this is what you want. Route messages normally when the user is online and trigger routing to your AS when he is offline.
