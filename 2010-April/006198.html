Anant,

> Now, I am trying to test calls involving preconditions using UCT IMS
> client.  I have configured "QOS Strength" on this client to "Mandatory".
> When I make a call from UCT IMS client, it sends an invite with
> preconditions to Open IMS. Open IMS is responding with "420  Bad
> Extension" with unsupported header set to "precondition".

Can you be more precise? And do you have a tracefile to see what you have
sent and which node created the response?

In principle when you use the tag "precondition" in a Require header field
it is the peer user agent which has to care about it, and if it cannot
support the precondition extension it has to reject the request with "420
Bad Extension". So in this case it is not the IMS platform which is
responsible to care for the extension but the other user agent.

This precondition extension should be transparent to the IMS core.
So I do not believe that it is the OpenIMSCore which responds.
