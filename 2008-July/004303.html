Yes, so you are not doing originating side triggering, but terminating 
side, which makes more sense for your service. Think about users from 
other domain calling in to your voicemail - you have no control on the 
service profile there.

btw: the empty profile means "match everything". I think this is what 
you want, but otherwise, write a TP that filters just what is required.

>>>> A few months ago Alberto has published a "how-to" for creating PSIs, see
>>>> below. Maybe this should be added to the FAQs (Provisioning Services).
>>>>
>>>> ---------------- How to provision distinct and wildcarded PSIs
>>>> -------------
>>>>
>>>> PSIs are implemented and work. There are several issues.
>>>>
>>>> Provisioning a PSI in the HSS is not easy, so you'll need to play around
>>>> a
>>>> bit.
>>>>
>>>> If you want I-CSCF routing is a bit easier and there wont be any S-CSCF
>>>> involved, this procedure is defined by 3GPP for trusted AS that don't
>>>> need
>>>> to go through the S-CSCF.
>>>>
>>>> I-CSCF routing
>>>> --------------
>>>>
>>>> - Put the address of your AS as a preferred-SCSCF (and set priority to 1
>>>> at
>>>> least)
>>>> - Create an IMSU for your AS and assign the preferred-SCSCF to it
>>>> - Create IMPI and IMPU  and attach the whole thing  IMSU-IMPI-IMPU
>>>>
>>>> UE--{SIP INVITE}---->I-CSCF----{Diameter LIR}--->HSS--{-Diameter
>>>> LIA}--->I-CSCF----{SIP INVITE}-->AS
>>>>
>>>>
>>>> S-CSCF routing
>>>> --------------
>>>>
>>>> - Create Application Server with your settings for the AS that hosts the
>>>> PSI
>>>> - Create iFC with no Trigger point (that means it matches everything) and
>>>> profile part indicator Any or Unregistered
>>>> - Attach iFC to AS
>>>> - Create a service profile and attach iFC created
>>>> - Create an IMSU and assign a normal preferred-SCSCF to it (but assign
>>>> one!
>>>> if not it wont work)
>>>> - Create an IMPI
>>>> - Create an IMPU choosing Distinct or Wildcard and write the wildcard in
>>>> the
>>>> appropriate format
>>>>    (sip:your*wildcarded?expre??ion at open-ims.test),
>>>>    It would be logical to click in the can Register thing and don't let
>>>> this IMPU to register and of course you have to click the
>>>>    PSI Activation to activate it and select the created service profile
>>>> for
>>>> that PSI
>>>> - Attach IMPU to IMPI and IMPI to IMSU
>>>>
>>>> UE--{SIP INVITE}---->I-CSCF----{Diameter LIR}--->HSS---{Diameter
>>>> LIA}--->I-CSCF----{SIP INVITE}-->S-CSCF---{Diameter
>>>> SAR}-->HSS--->{Diameter
>>>> SAA}-->S-CSCF---{SIP INVITE}-->AS
>>>>
>>>> --------------------------- End of How-To
>>>> ----------------------------------
>>>>
>>>> I hope that helps.
>>>>
>>>> Franz
