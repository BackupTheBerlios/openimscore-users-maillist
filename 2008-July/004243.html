Hello Franz,

I've detected this problem as well, but I think the Authorization header 
is not really being truncated, but Wireshark uses that "notation" when 
the line exceeds the maximum number of characters allowed, but it's only 
a GUI issue. If you expand that line, you can see all the fields and I 
think nothing is truncated.

I think the problem is that ims-communicator doesn't include the qop 
related parameters in the Authorization headers, and then the 
registration is rejected. I'm having the same problem and I think only 
OpenIC Lite 1.2 is able to include that parameters. This is new in OSIMS 
since r567 and even when some problems related to this issue have been 
solved since r570, only clients which manage qop parameter can register 
(AFAIK).
I don't know if it's possible to disable qop during registration to let 
old clients to register...

Regards,
Jose
