Hi Sabyasachi,

Hmm... I don't know... I think that the server-side mechanism that we 
have now is better and lighter, cause you don't have to rely on the 
client. There are many clients out there and it's hard to require all of 
them to support this feature. We're rather going for an P-CSCF 
help&enforce way. So, in no case we will replace completely the current 
feature that sends at regular intervals pings from the P-CSCF to the UE 
(you can though disable it from configuration as you wish). So far, 
together with the RTPProxy, we dodged quite well most of the NATs...

If we'd see it as an extension, there's no downside to supporting this. 
Yet it is a little bit out of scope here as it should be implemented in 
the lower layers of SER (see iptel.org for the underlying software for 
our CSCFs). So maybe a request there would be better targeted.

In any case, we would gladly accept such help/patches even just here, as 
it is an obvious improvement. I for one though, realistically speaking, 
don't think that I have enough best-effort time slots in the next weeks 
to implement it myself.

Cheers,
-Dragos
