Hi Reza,

> I got this when executing pcscf :
> 
> 
> root at hire:/opt/OpenIMSCore# ./pcscf.sh
> ./pcscf.sh: line 4: setkey: command not found
> ./pcscf.sh: line 5: setkey: command not found

These errors are caused by the start script. I guess that you do not have
ipsec-tools installed.

>  0(17878) parse error (26,23-25): syntax error
>  0(17878) parse error (26,23-25):  hostname expected
>  0(17878) parse error (26,25-26):
> ERROR: bad config file (3 errors)

There is something wrong in the config file pcscf.cfg in line 26.
This is the alias statement for the hostname of your P-CSCF. 
It should look like: alias=pcscf.net1.test:4060

I again guess that you have not fully changed the domain names in all
relevant files.

> executing ping pcscf.open-ims.test seems to be ok, but ping
> pcscf.myname.com somehow doesn't work well.

Why do you have two domains?
Either you use the original configuration data, than you have
pcscf.open-ims.test, but if you have changed the domain you only should have
pcscf.myname.com.

Did you also modify your DNS accordingly?
