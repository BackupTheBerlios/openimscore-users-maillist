Hi Khaldon,
 
> Asterisk has a lot of really good functionalities. For instance, Asterisk
> can communicate with 3G networks if you install a ISDN card
> (http://www.voip-info.org/wiki/view/Asterisk+H324M).

Yes I know. That's not the way foreseen by the standards where SS7
interconnection should be used, but for small environments (e.g. testbeds)
it could make sense. 

> Many other applications can be offered (Video IVR...).
> That's a pity that no one tried to interconnect Asterisk with OpenIMSCore
> more seriously. May be we should develop a proxy server that convert SIP
> to IMS and vice versa. It will be helpful for the NGN community.
> 
> What do you think about it?

Yes that's a good idea and from my point of view not so hard to do.
One needs some know-how on attaching applications to the IMS some skills on
how to manipulate with header fields with the help of SER or OpenSER. That's
it. 
One should do it systematically: application for application.
Maybe some applications are easy to adapt and others maybe more difficult,
but in fact an interesting approach.

Nice idea.
