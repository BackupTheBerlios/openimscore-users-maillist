Hi,

Assuming you have a service that wants to save this information you can use
the Sh-interface [1] (based on Diameter) for that. Data can be stored per
IMPU or PSI; in your example I'd go for per IMPU.

Use Sh-Update to store the data and Sh-Pull to retrieve it later.

PS: A trick to find similar information by yourself in the future. First,
have a look at [2] and locate the nodes you're looking for. Next, lookup the
interface defined between these nodes. If there is no direct interface
between them look at intermediate nodes and see whether you can use those
instead. 

In this case, start by looking at Fig. 6 in [2] (it is even easier looking
at the following figures, especially Fig. 6b). Locate the HSS and the AS
entities and see if there is an interface between them. Luckily, in this
case we have the Sh-interface.

Then, we need to find out what this interface is about. First place to look
at is Section 6a.7 in [2]. Check either the TOC or just glimse through the
document and you should find section 6a.7.16. (A more brute-force method
here is to look at the 3GPP TS 29-series (either
http://www.tech-invite.com/Ti-ims-standards-29.html or
http://www.3gpp.org/FTP/Specs/html-info/29-series.htm) and search for "Sh"
in the title of those. The reason for looking at the 29-series is that we
know it should be part of "Signalling protocols ("stage 3") -
intra-fixed-network").

Anyhow, in that section you get an "executive summary" of that interface and
a pointer towards more information, which is [3] subclause 4.2.4. In
particular, section 4.2.4a gives you more details. Strangely enough, I
cannot find a reference from this section to [1], so it looks like you'll
need to apply the "brute force" method described above in any case.

[1] 3GPP TS 29.328.
http://www.tech-invite.com/Ti-ims-standards-29.html#ts-29328
[2] 3GPP TS 23.002.
http://www.tech-invite.com/Ti-ims-standards-23.html#ts-23002
[3] 3GPP TS 23.228.
http://www.tech-invite.com/Ti-ims-standards-23.html#ts-23228
