I find this to be an interesting thread. We have an asterisk + SER
installation at our university, and I always thot it an elegant
solution to integrate asterisk with Mobicents using the asterisk RA,
and then hook up the mobicents as an AS for IMS. The advantage is that
JSLEE is being marketed as a highly efficient platform for handling
hundreds and even thousands of SIP transactions a second. We teach
appication development in asterisk using the usual tool like manager
interface and AGI, but I am coming under the increasing opinion that
asterisk should be used as a switching core and applications
should/could be hosted elsewhere, with asterisk as a routing engine
for selecting the appropriate application server to provide a service
to a requesting user. Any ideas on this?

Mos
