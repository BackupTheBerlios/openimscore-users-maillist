Hi Suresh,

> Somewhere in document for SER, I have read that we cannot
> CONTROL(terminate, etc.) an ongoing session through SER i.e. in other
> words we cannot have prepaid billing implemented through SER as base.

SER acts only as a SIP proxy server. Therefore it cannot actively terminate
a session.

> Is it that we cannot implement online charging in SER_IMS as it is based
> on SER?

No. In an IMS architecture you can use application servers which act as
B2BUA. These can terminate sessions.

> Or if we CAN, then do we have to have very tedious effort for doing it
> through SER_IMS? Can you comment on the time-line, learning curve for
> doing the same.

You only need to setup an AS acting as B2BUA and develop the application
using e.g. SIP servlet technology.

> and  also can some one let me know How to enforce call tear down when
> credit is exhausted in SER_IMS.

As mentioned above: with the help of application servers.
