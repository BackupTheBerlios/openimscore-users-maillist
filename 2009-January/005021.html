 >Are policy control over Go interface implemented in openims?

No. OpenIMSCore only includes the functinoalities P-CSCF, I-CSCF, S-CSCF 
and HSS.
Go interface is a 3GPP Release 6 interface between PDF and PEP (in GGSN)

Even though the specifications says that the P-CSCF and PDF could be one 
common entity we do not support this option. We do support a Release 6 
Gq interface towards a PDF.

 >Are changes in IMS for packet cable qos done in openims?

No. I am not sure which changes do you talk about but according to 
PacketCable 2.0 its PacketCable standards the one that will converge to 
3GPP approach for QoS provisioning based on the PCC architecture.

The PacketCable Application Manager implements a Release 7 Rx interface 
towards the P-CSCF. This interface IS implemented in OpenIMSCore. 
Changes in the PCC architecture in order to support the DOCSIS IP-CAN in 
the Rx interface as explained in TS 23.203 Annex D and TS 29.214, if 
any, have not been implemented.


 >Are changes for firewall and nat traversal done in openims?

There is support for NAT in the OpenIMSCore using optional functionality 
that will modify SDP contents in SIP requests and responses.

 >Can you tell whether GRRU, policy server, packetcable application 
Manager,supported in openims?

GRUU (Globally Routable UA URI ) is not yet supported in OpenIMSCore 
although there has been interest on it before.

Policy Server and Policy Application Manager with a valid Rx interface 
towards the P-CSCF as specified by PacketCable 2.0 QoS specification 
should work with OpenIMSCore. The PS and PAM themselves are out of the 
scope of the OpenIMSCore project but if you have one and have 
interoperability problems with the P-CSCF Rx interface we would be 
interested in your contribution to the project to make it more 
interoperable and better
