Hi Suresh,

from the incomplete trace I see the S-CSCF triggering towards your AS: 
172.18.3.52:5065. Then I see that your AS does some kind of weird B2B 
(same call-id, different CSeq) back. However, your Request-URI is wrong 
as there is no user sip:bob at 172.18.2.187:6060 in your network. The user 
was probably sip:bob at open-ims.test...

So, the error message is right - "or terminating user not suitable for 
unregistered services" - obviously as there is no user profile for that 
user with a unregistered state IFC. Please keep in mind that the S-CSCF 
can only process requests for originating or terminating users that it 
knows.

Cheers,
-Dragos
