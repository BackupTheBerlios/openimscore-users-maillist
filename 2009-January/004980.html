Hi Suresh,

> My questions are:
> Q1: Do I have both options for implementing online charging i.e.
> option 1: IMS itself handles call control with help of functionality
> similar to release_call.
  
This option is not in-Line with IMS architecture. The situations where
sessions are terminated by P-CSCF or S-CSCF are caused by standardised
behaviour of these network elements.

> option 2: With help of new charging AS (b2bua form) call control is
> implemented.

This is the way conformant to the IMS architecture. Using option 1 would
mean that you are modifying P-CSCF or S-CSCF behaviour. Applications have to
be implemented in an application server otherwise you are violating the IMS
architecture.

> Q2: If I have both options, are there any differences w.r.t.
> performance, capabilities, etc between the two options

The main difference is: With option 1 you are destroying the IMS
architecture.

> Q3: (A bit redundant question) Why option of B2BUA was suggested in
> place of "actually modifying the IMS for call control" (of course if
> both options are similar)

As mentioned above: because of the defined architecture.
