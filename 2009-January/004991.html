Hi Erling,

well, for the MSISDN via UDR, you are right: it is a stub so far as we
did only look at IMPUs (read: SIP or tel URIs) for the HSS and left out
the HLR-related "Basic MSISDN" values. 

The DSL Line Identifier is currently only used for NASS-bundled
authentication scenarios and nothing more.

I am not sure if we should mix those two values up but a fix would to
add the option to explicitly link MSISDN information to an IMPI to the
FHoSS.

Regards,

Peter
