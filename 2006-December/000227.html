Hi Franz,

No, it is correct. Each initial request should follow the Service-Route 
indicated at registration. That Service-Route contains for now the 
S-CSCF that is responsible for applying the service profiles for the 
originating user and also do all the origination related operations. The 
normal path of the INVITE would be: 
UE1-PCSCF1-SCSCF1-[ICSCF2]-SCSCF2-PCSCF2-UE2

George is not actually using that service-route as we can see in the 
trace. Fortunately the P-CSCF is set to enforce the Service-Routes by 
adding the value that it had seen at the registration and the call still 
goes on. In the pcscf.cfg you can use another variant, to refuse 
forwarding of messages not following the Service-Route, like it happens 
now with the Record-Routes.

-Dragos
