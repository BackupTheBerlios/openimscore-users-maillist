Hi Babu,

This seems to be the same problem that several of us already faced
earlier (see
http://lists.berlios.de/pipermail/openimscore-users/2006-December/000169
.html
http://lists.berlios.de/pipermail/openimscore-users/2006-November/000124
.html) except that in those cases it was due to "compatibility issues"
between UE and sip2ims gateway. If you are using an IMS UE and directly
sending to the P-CSCF, the root cause is probably slightly different.

Anyway, I think the issue is related to the following debug output:
	1(8914) >>       Orig_Subsequent: Request not following
indicated 	dialog routes

If you look into pcscf.cfg, you'll see that it is printed when the
P_follows_dialog_routes() check fails. And this happens when the Route
headers received by the P-CSCF for the ACK does not match what the
P-CSCF expects. In my case, the top most Route was missing.

Hope this helps,
-David
