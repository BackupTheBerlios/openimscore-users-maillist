Hi
I just wanted to mention that both Relase 6 and 7 of 24.229 says how third
party REGISTER should be supported.
See 5.4.1.2.2 "Protected REGISTER", bullet 13) and 5.4.1.7.

The problem of using the Sh interface in order to receive service profile
updates, is that you have to make a subscription
for each possible user in advance, which is not very nice if you have a lot
of them.

Could you give any indication of what part of the code that has to be
modified in order to implement third party REGISTER,
and the complexity of it (besides t_replicate()).

By the way I think that the Open IMS is a brilliant idea and the core seems
very promising so far.
We are using it in order to test our Application Servers (IM and Presence).

Erling
