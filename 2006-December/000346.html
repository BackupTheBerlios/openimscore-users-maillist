Hi all!

When I start all IMS Core components I cyclically get the following error messages
from the P-CSCF:

 4(23171) ERROR: sip_hostport2su: could not resolve hostname: "imscore"
 4(23171) ERROR: uri2dst: failed to resolve "imscore"
 4(23171) ERROR: t_forward_nonack: failure to add branches

It seems to be caused by the SIP2IMS gateway in the following context:

SIP2IMS-Trace:

Authorization: Digest username="alice at open-ims.test@imscore:3060", realm="imscore",
===
 1(23291) INF:SIP2IMS:sip2ims_get_public_identity: <sip:alice at open-ims.test@imscore:3060>
 1(23291) INF:SIP2IMS:Gw_MD5_to_AKA:
=== 

* I'm not running any SIP client at that time!

* My /etc/hosts contains a valid hostname "imscore" but I used the real IP address (not 
  localhost or 127.0.0.1) in all configuration files.

* My /etc/hosts

  # Do not remove the following line, or various programs
  # that require network functionality will fail.
  127.0.0.1       localhost
  139.2.247.52    imscore
  139.2.247.52    scscf.open-ims.test
  139.2.247.52    icscf.open-ims.test
  139.2.247.52    pcscf.open-ims.test
  139.2.247.52    hss.open-ims.test
  139.2.247.52    imscore.open-ims.test
  139.2.247.52    sip2ims.open-ims.test

What kind of authorization does the SIP2IMS gateway and where comes the (strange?) 
digest username "alice at open-ims.test@imscore:3060" and the used realm "imscore" come 
from? And why can "imscore" not be resolved my P-CSCF?

Thanks in advance,
Sven
