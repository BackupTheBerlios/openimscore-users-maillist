Hello Ben,
You can generate HA1 by using:
./gen_ha1  user at open-ims.test   open-ims.test  password
 
HA1B:
./gen_ha1  user open-ims.test password.
 
Use these values and see if it works.
 
Following is from the FAQ on OpenIMSCore.
SIP-to-IMS Gateway

*	How do I provision a new user? 
	Use serweb or another tool to create a user in the credentials
table. Alternatively, you can manually add a similar entry to those of
Alice/Bob. HA1 is irrelevant and can be left blank. HA1b can be computed
using e.g.: 
	ser_ims/scripts/gen_ha1 user realm password
	Don't forget: The username must be the same as the private AND
the public identity of the user, as SIP UEs do not usually make a
distinction between them. 

also check the following discussion regarding the same problem that you
see when generating HA1b
http://lists.berlios.de/pipermail/openimscore-users/2006-November/000078
.html
 
I added a new user successfully by only using HA1B generated by the
script, I did not leave HA1 field blank, but used the same as for user 
alice at open-ims.test.
 
Check if it works for you
 
Regards,
Gaurav

Hi Ben,
Follow these steps to add a user ben at open-ims.test
<mailto:ben at open-ims.test> 
1. Go to /opt/OpenIMSCore/ser_ims/utils/gen_ha1/
2. generate ha1b ./gen_ha1 ben open-ims.test ben 
adebaa5b679f75d7e19b3a7a118f35e0
3. Add user ben at open-ims.test <mailto:ben at open-ims.test>  in sip2ims.sql
similar to bob/alice. Use the ha1b generated above for the new user.
3. Go to /opt/OpenIMSCore/FHoSS/scripts/userdata.sql
4. Add the new user similar to alice/bob.
5. Update MySql with the new userdata.sql and sip2ims.sql

Gaurav
