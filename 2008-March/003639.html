Hmm... true, but I think that it was really meant by 3GPP in this way. 
If I remember well the Registration Type was introduced at a later phase 
and as such, for backwards compatibility matching all REGISTER requests 
is represented by the lack of any indication for RegistrationType (also 
this is efficient as the 0 element case can not represent an empty set 
as this will not ever match). So if it's any then you should have 0 
elements. To prevent different interpretations, I would leave it like it 
is, as this is the 3GPP published xsd.

> to 3.
