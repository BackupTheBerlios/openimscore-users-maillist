Hi Franz,

>Why do you think of high load? This is exactly one REGISTER transaction
triggered as a 3rd party registration from the S-CSCF towards your AS.
>Do you know that you can avoid to get Re-REGISTER requests in your AS
by setting the iFC appropriately?

Yes, I know about IFC, but if we have 1 000 000 users and need
notfications about registation of particular 10 000 users, than it will
be a bad idea to handle REGISTER requests from all our users.
For example, imagine a following scenario: user A sends message to user
B while user B isn't registered. We can store this message and resend it
to user B when he/she will register. So we need to know somehow when
user B will send REGISTER request, but it has no sense to handle
REGISTER requests from all other users as it will cause extra load on
AS.
