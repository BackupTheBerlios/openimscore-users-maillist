Hi Marco,

> I've checked both the IMS client and AS (IMS's side) configurations, but
> I don't understand where can I set support for "sec-agree" requirement
> (I understood that it was the precondition to set, isn't it?) or where
> (in the IMS client) to set to avoid precondition requirements.

When the client sets "Require: precondition" than it means that it requires
SDP preconditions to be supported by the session partner which is Asterisk.
Asterisk does not support it and therefore session setup is not possible.

The only way to avoid the problem is to use a client, which does not require
preconditions, e.g. X-lite. Or does UCT IMS client allow to configure
preconditions?
