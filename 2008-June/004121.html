Andreas Häber wrote:
> Sending the new REGISTER-request to P-CSCF should be pretty similar to how
> you sent the initial request. Just be careful to send it in the same dialog,
> instead of creating a new dialog.
>   
Errm :), although this looks correct, it's not. This is because there is 
no such dialog defined for REGISTER. Actually, the RFC says something 
more like: resend the original message including a response to the 
received challenge, in an Authorization header. So, it should have the 
same Call-ID and most of the original header, also different in CSeq so 
that it won't be mistaken for a restransmission of the first, but in the 
same dialog it's too much to say... Dialogs have additional recorded 
Routes and the 2nd REGISTER is not as such subsequent and following the 
original path (although this would be pretty cool, considering all the 
duplicate operations that will happen on the path).
