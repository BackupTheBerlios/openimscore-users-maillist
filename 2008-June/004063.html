I think that Franz referred to using different REGISTERs for each 
contact, as he said to use different IMPIs (You can not use 2 IMPIs in 
just one REGISTER). This would've worked around the bug.

However, for scenarios where you would like to hand-over your current 
sessions from one AN to other on the same terminal, it has to be 
possible to use the same IMPI with different contacts. And I don't think 
that we have now such a restriction... (or if it is, it's a bug :-p).
