Thanks for the trace I was interested in. Unfortunately it did not show what
I expected.  

First there are some packets missing (e.g. the INVITE from P-CSCF to S-CSCF,
between packet 26 and 28) and some packets are fragmented. You can avoid
fragmentation by using tcpdump with a negative filter e.g.:
tcpdump -i any -s 3000 -w /var/tmp/trace.pcap not port 22 and not port 3306

Next point is that I am sure that your call flow is not IMS conform.
The INVITE of the client addresses the AS directly with a numeric IP address
(INVITE sip:100050 at 10.107.109.107). 
If you include an AS in an IMS network you should use iFCs and PSIs (Public
Service Identities). This I had expected. Maybe you should try it in that
way.
