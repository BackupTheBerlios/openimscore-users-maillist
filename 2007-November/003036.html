Hi all,

I'm having the following problem trying to registering an IMS user agent in
my testbed: 403 Forbiddent Hss unable to comply.
I will try to explain briefly what the structure of the testbed (core ims)
and what is happening:
- P-CSCF is installed in one machine (10.14.1.2)
- I-CSCF, S-CSCF and HSS installed in same machine (10.12.1.5);
- The listenning ports that are the one defined in the initial configuration
files;
- An IMS user agent installed in another machine (10.16.100.6);
- The version of the HSS is last one provided in the SVN;
- DNS and reverse DNS lookup is correctly working (open-ims.test domain);

I started all components and tried to register the following user
Bob at open-ims.test; The  Register message sent by the user agent reaches
P-CSCF that forwards it to the I-CSCF. The I-CSCF sends a UAR to HSS that
Replies with a UAA (that contains an result code=
DIAMETER_UNABLE_TO_COMPLY). So the the I-CSCF sends as response to the
Register SIP message an 403 Forbiddent Hss unable to comply;
We have looked to HSS code and it seems that HSS only answers this way when
in the diameter request (UAR) does not provide the
Visited_Network_Identifier avp, but as we can see in the ethereal capture
this avp is provided.
In attachment are the logs that I have (HSS, I-CSCF and ethereal capture).
Can anyone help me solving this issue?
