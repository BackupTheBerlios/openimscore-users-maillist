Hi,

We are using the OpenIMS Core in our testbed intensively - thank you for
the great product.

Within the P-CSCF all the dialogs are checked against expiration in the
function dialog_timer() in modules/pcscf/dlg_state.c. However, the
dialog will only be removed if the dialog direction is
DLG_MOBILE_TERMINATING. Therefore, DLG_MOBILE_ORIGINATING dialogs are
never removed if they are expired and the dialog list rises forever.

Is the differentiation of the dialog direction necessary? I guess all
expired dialogs should be removed anyway, shouldn't they?
