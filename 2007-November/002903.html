Hi all,

I have just upgraded the ISC modules in r490.

- the public identity is saved in the ISC mark => now we don't care
anymore about the S-CSCF changing the R-URI or dropping the
P-Asserted-Identity header
- introduced new timers just for fwding on the ISC interface
   - isc_fr_timeout/isc_fr_inv_timeout
   - before the overall fr timers were set for the S-CSCF
   - now non-ISC transactions won't timeout prematurely as before

About the timers... I have the feeling that once the AS responded, the
transaction timers should be re-extended to their normal SIP values.
However, nobody complained about it before, when it was even worse, so I
have doubts. Also, it is a little difficult as I can not distinguish
between the first and next replies, so I have some trouble extending it
just once.

I have tested it a little, but it needs much more testing. So pls pls
pls, if you have time to spare send me feedback if I broke or not your
use of the ISC interface. Please provide also traces if you report bugs.

Thanx,
-Dragos
