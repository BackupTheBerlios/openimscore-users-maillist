Hi,

I thought the detection of changed Request-URI was related to usage of
B2BUA ASs and the need to include an "original dialog identifier"
(ODI) in the ISC mark in the Route header in order to detect this
Req-URI change and apply different routing logic, as in 24.229 r7 and
8.

Just including the public identity in the mark is not enough for this
use case. We need to compare the Call-ID, from and to tags (dialog id)
in a message coming from an AS with the ODI in its Route header and
check if this ODI is stored in the S-CSCF.
Furthermore, in the general case ODI may be an actual SIP dialog id,
or a SIP transaction id (think of triggers for MESSAGE going through a
B2BUA AS).

We've been working on support for ODI but it's not yet complete.
