Hi David,

Unfortunately that branch is pretty much abandoned. The developer left
before he could finish it so it is incomplete and very buggy. Plus that
the code there is kind of deprecated. You see, initially when I started
CDiameterPeer stack I have just put there a stub for basic session
support. Then 2 people started in parallel development on the Auth and
Acct session support, but unfortunately they did not work together, nor
did they use the basic session stub, so there were 3 parallel session
implementations.

Recently, Alberto picked-up the Gq/Rx work and we revamped the session
support - in the 08 branch. So far we did the basic data structures, the
stateless auth state machines and the statefull client auth state
machine. Similarly the accounting state machines need to be developed
and then the old Rf code converted to the new API.

Unfortunately me and Alberto do not have any resources to spend on the
Rf support for now. But, in case you feel like writing some code, we
will gladly help you.

Cheers,
-Dragos
