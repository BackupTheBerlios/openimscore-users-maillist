> I still believe Monster is the problem here. 

> The port number is 5060, it just gets add to the IPv6 address, and P-CSCF
just
> extracts information and nothing more.

I think we both are right. There is a problem at Monster which I did not see
before: It uses an erroneous address in Contact header
"[3ffe:200:58:1:0:0:0:2:5060]" by adding the port-number to the address
part. In the Via header field the address is correct:
[3ffe:200:58:1:0:0:0:2]:5060, the port-number kept outside of the address.

But in addition also the parser in OpenIMSCore is erroneous, it sometimes
truncates parts of the address before storing it in P-CSCF. 

So there are 2 problems:
- Monster creates an erroneous Contact header field.
- OpenIMSCore parses sometimes  the address in Contact header field
erroneously.

For the parser problem in OpenIMSCore I just looked into the parser code
(module ser_ims/trunk/parser/contact/parse_contact.c) and I found that there
is no parsing for IPv6 addresses.
