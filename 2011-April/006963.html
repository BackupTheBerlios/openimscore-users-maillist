Hi Jason,

> What is the process for routing an INVITE to a voicemail App Server (AS)
> when the called B party is offline?

This is called "Mobile Terminating call procedures to unregistered Public
User Identities" and is specified in 3GPP TS 23.228 §5.12.

> My current understanding is that this should be done using IFC
(terminating
> user unregistered). BUT, without a registration from the B party - the
S-CSCF
> and I-CSCF won't have any IFC information (Service Profile) for the B
party.
> Does this mean that the S-CSCF or the I-CSCF should request a service
profile
> using the public ID of the B party to pull service profile information?
This is
> the only way I see this flow working? Or am I missing something?

If no S-CSCF is already assigned, it is selected and assigned at the very
moment when a terminating request arrives. Then the iFC for terminating
unregistered services is executed.
