Hi,

 

a simpler explanation:

It allows roaming. When a user tries to register in a different network than
its home network,  that means the P-CSCF is in a visited network, then the
P-CSCF adds the P-Visited-Network-ID header field to the REGISTER request
and forwards the REGISTER request to an I-CSCF of the home network. The
I-CSCF then queries the HSS among other things if the user is allowed to
roam in the visited network.

If not it rejects the REGISTER request.
