Dear list,
I have a research project involving distributed load-testing of Open IMS
core with IMS Bench SIPp.
I have a Gentoo amd64 quad-core 4GB RAM server with Open IMS Core
installed from SVN in November 2010.
DNS is external, loaded with a real public domain.
I'm testing from home over a 4Mb/s WiFi link, IPv4 and NAT. IMS_Bench is
installed from source.
I have loaded some 24000 users into the database with the supplied
srcipt. I can register with an IMS client and call between two of them.
IMS Bench scenarios seem to work as well.

However, the server is terribly unstable. When I start registering users
at a rate of 100 reg/s, the S-CSCF crashes nearly immediately. Worst is,
even if I load it very gently, with 10 reg/s, CPU load is negligible,
but after a few thousand of registrations, the S-CSCF locks up. I never
achieved more than 3700 regs at once. Restarting S-CSCF helps.
Symptoms:

SIP message: SIP/2.0 500 Error forwarding towards next S-CSCF

I-CSCF console:
4(22269) ERROR: uri2dst: failed to resolve "scscf.open-ims.test"
4(22269) ERROR: t_forward_nonack: failure to add branches
4(22269) ERROR: w_t_relay (failure mode): forwarding failed

S-CSCF console:
4(22241) ERROR:get_routing_peer(): No connected DefaultRoute peer found
for app_id 16777216 and vendor id 10415.
4(22241) ERROR:AAASendRecvMessage(): Can't find a suitable connected
peer in the routing table.
4(22241) ERR:S-CSCF:S_challenge: Error retrieving an auth vector

I am pretty sure, that there is no ".open-ims.test" left in the config
files or database. Only in the source code.

Has anyone encoutered that? What should I do?
Thanks, Tomas Vondra
