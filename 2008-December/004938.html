I've had a peek inside the sql files and it seems that the mysql permissions are being set for (userx)@localhost. The configurator script only replaces the ip address, so even after running it the db permissions will still be for local usage only. Your cscfs will be trying to connect from non-loopback addresses. Try replacing the @localhost with @192.168.x.x before reloading the sql files.

> ims.test not responding to connection attempt.

> Please advise.

There is a script doing that job: configurator.sh

These are the instructions:
In order to change the domain name and ip address you can use a
script(configurator.sh) at ser_ims/cfg/. Copy the all *.sql *.cfg *.xml
files that you want to modify to the same directory as configurator.sh and
execute it. Follow the instructions then you should have your files
reconfigured for new domain and ip.

My experiences: apply it in the following directories:
- /ser_ims/cfg
- /FHoss/deploy
- /FHoss/scripts

And then it should work.
