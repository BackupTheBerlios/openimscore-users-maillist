Another point: I faced this problem today as well, and INVITE goes out and I
get a 408 Timeout. Have a look at your INVITE - does it have a Service-Route
header? If yes, is it the same as the one you received in the 200 OK for REG
? If not, it should be corrected. Otherwise, you can configure P-CSCF to
ignore the Service-Route header check like this:

 if (!P_follows_service_routes()){
               #Variant 1 - deny access to the network
               sl_send_reply("400","Bad Request - Not following indicated
service routes");
               break;
               #Variant 2 - enforce routes and let the dialog continue
               #P_enforce_service_routes();
SHOULD  BE CHANGED TO:



 if (!P_follows_service_routes()){
               #Variant 1 - deny access to the network
               sl_send_reply("400","Bad Request - Not following indicated
service routes");
               *#break;*
               #Variant 2 - enforce routes and let the dialog continue
               *P_enforce_service_routes();
