Hi Raghu,

> Has anyone encountered this before? Do you have a reason for this?

I guess that you have not disabled the check for the Service-Route header
field.
You have to enable Variant 2 in pcscf.cfg:

        if (!P_follows_service_routes()){
                #Variant 1 - deny access to the network
                sl_send_reply("400","Bad Request - Not following indicated
service routes");
                break;
                #Variant 2 - enforce routes and let the dialog continue
                #P_enforce_service_routes();
        }
