:) I get that feeling too some times.

It is important to think of several issues for priorities:
- do the trigger points overlap? might they match the same thing? if 
they do, you clearly need to think about which one you would like to 
handle first. Should you do charging before the voicemail, for example?
- if one AS terminates the call (sends a final response), then the ones 
with lower priorities will never be triggered. So maybe you need to put 
the AS working in proxy mode before the ones in termination mode... But 
this also depends on the service.
