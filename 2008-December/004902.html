For the first thing: ERR:I-CSCF:cscf_get_private_identity: No 
credentials for this realm found.

I can't really say what is wrong... The private identity and the 
integrity protected are extracted in the I-CSCF from the Authorization 
header. The realm is extracted from the Request-URI and based on the 
trace, it should work just and you should not get that error

For the 2nd thing: ERR:S-CSCF:S_is_authorized: no matching auth vector 
found - maybe timer expired

This is because the client tries to reuse the authentication vector and, 
AFAIK, for AKA they are not to be re-used. So the S-CSCF can not match 
it as it was dropped when the response in the initial registration was 
received. Anyway, a new vector is requested from the HSS and a challenge 
is sent. Then you might have something wrong in the AKA implementation 
on the client as it fails to produce a proper response in the last 
REGISTER request.
