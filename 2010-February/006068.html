ngrep is exactly what you want. Probably also wireshark can export in a 
text format, otherwise I can't explain how some people paste those ugly 
traces into emails when attaching the pcap would've been better.

And for capturing all the traffic between the P/I/S-CSCF you'll need to 
capture on lo, because the kernel will route it internally.

>> The most comfortable way to dump the signalling messages is using 
>> tcpdump on

>> I typically use the command:
>>  tcpdump -i any -s 3000 -w /var/tmp/trace.pcap not port 22 and not port
>> 3306
>> Via this command I exclude SSH and MySQL packets.
