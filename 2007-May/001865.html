Hello,
We have found that S-CSCF in some cases does not send some _mandatory_  
AVPs in Push-Profile Answer (PPA) Command -  i.e. 
(1)Vendor-Specific-Application-Id, (2) Auth-Session-State. This happens 
every time we send a PPR command on a user (IMPI) that is not registered 
on currently serving server (registration procedure was not performed 
before). And because the missing AVPs are mandatory, we get an 
unexpected error and cannot handle this event appropriately.
In the attachment I'm sending a wireshark capture and log files from 
icscf, scscf and pcscf.
