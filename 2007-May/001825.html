Hello Dragos:
    I found in P-CSCF there is a possible bug: in security.c
save_contact_ipsec function, when call update_r_contact_sec, the parameter
you use for the host port is ipsec->port_uc. But I think it should be
ipsec->port_us. Because according to the standard, by UDP ipsec, the
response and request to the UE shall be send to the UE port-s instead of
port-c. Also when UE send request, it shall set its via header to the port-s
instead of port-c. According to the currently alghorithm in P-CSCF to find
the contact information, it use via header in the request to find the
contact. So if the client is corrent(set via to port-s), the contact can not
be found(because in current P-CSCF, it is saved with port-c)!

    could you please check the code and standard and confirm about my
investigation? thanks a lot!
