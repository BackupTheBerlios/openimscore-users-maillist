Hello!
 
I am doing some tests with Xlite and Sofia SIP client with these
configurations:
testbed 1: xlite client (bob) <-> ompen ims <-> xlite client (alice)
testbed 2: Sofia SIP client (bob) <-> ompen ims <-> Sofia SIP client
(alice)
 
The 2 clients are:
Xlite v3.0 build 34025
Sofia SIP CLI v0.12 and SOFIA SIP 1.12.5
 
Testbed 1
Registration OK
Instant Messaging NOT OK
 
Testbed 2
Registration OK
Instant Messaging OK
 
Giving a look to the message snoops I saw this:
Testbed 1
Bob sends a register, bob receives a 401 unautorized, bob sends a
register with the right token, bob receives 200OK and the registration
is OK.
Bob sends (automatically and in a periodic way) a subscribe to
bob at open-ims.test and the event in the subscribe is message-summary.The
response is: 408 Request timeout
Bob sends a message to alice with header content type:
application/im-iscompsing+xml.The response is: 408 Request timeout
 
Testbed 2
Bob sends a register, bob receives a 401 unautorized, bob sends a
register with the right token, bob receives 200OK and the registration
is OK. 
Bob sends a message to alice with header content type: textplain.The
response is: 200OK and alice receive the message.
 
The only differences in the IM is the field content type.
 
 
Have you done any other test with IM ?
Are there any client supporting IM ?
