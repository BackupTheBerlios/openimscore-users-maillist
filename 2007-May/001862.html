Hello everyone,

I'm trying to setup IMS Core to study roaming case. However I'm stuck with 
DNS configuration.


My current network is:


[ims.foo domain] ------- public inter domain (foobar.foo) ----------- [ 
roaming.ims.foo domain]
  private DNS                              public DNS              private 
DNS

How can I set up private domain in let's say roaming.ims.foo to register 
alice in home network (ims.foo)

For the moment both networks work fine seperately. So the DNS 
configuration maps NAPTR and SRV requests to icscf.ims.foo or 
icscf.roaming.ims.foo

What I am trying to do is that when alice registers in roaming.ims.foo 
with ims.foo uri, my NAPTR and SRV records would return 
accessrouter.foobar.foo domain (public domain).
Afters a simple DNS A request would gibe me IP of accessrouter.foobar.foo 
and register would be directed towards ims.foo domain.

The problem is:

1) The DNS in roaming.ims.foo does not seem to give back 
accessrouter.foobar.foo when ims.foo SRV request is sent. It just forward 
the request to public DNS which of couse does not know ims.foo

Question:

1) Can this case work ? i.e. interconnecting 2 privates domains via a 
public domain. e.g. two private sites connected via internet.
2) If yes to 1) how should I configure DNS in private domains to make this 
domain name translation/mapping i.e. the dns in roaming.ims.foo receiving 
a request for ims.foo and replying accessrouter.foobar.foo

My current configuration is: in private DNS (roaming.ims.foo) I configured 
db.roaming.ims.foo with A, SRV and NAPTR records for local requests. For 
roaming (same DNS) - I also configured db.ims.foo with SRV and NAPTR 
records with the translation line : _sip._udp.ims.foo. SRV 10 10 
accessserver.foobar.foo



Can you help me please or give me some interesting links
