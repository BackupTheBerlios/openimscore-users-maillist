> There must be a commuication problem between FHoSS and mysql somewhere ...

Yes, I am not a heavy MySQL expert but it seems that there might be a
problem with user privileges.

The last line in hssdb.sql is:

grant all privileges on hssdb.* to hss at localhost identified by 'hss';

I remember that I had a similar problem which I solved by an additional
MySQL statement which I inserted once:

grant all privileges on hssdb.* to hss at 192.168.0.7 identified by 'hss';

192.168.0.7 is the address of hss on my platform.

Maybe that helps.
