Hello,
I'm trying to set up the following testbed.
Openims core platform installed on machine A which has the IP address 172.31.77.140.
One client reaching network interface of A from another machine B.

Now what I've done is the following:
-adjusted pcscf.cfg, scscf.cfg, icscf.cfg with the line
listen=127.0.0.1 172.31.77.140
to make them listen not only on loopback interface but also on the network interface (172.31.77.140)
(I've verified and now ports 4060,5060,6060 are opened both tcp and udp on both ip addresses)

-adjusted the dns server used by machine B with an entry pcscf.open-ims.test which resolvs to 172.31.77.140,
also there is an alias open-ims.test to the same 172.31.77.140

The machine A with openimscore platform uses a locally installed DNS server which resolves every openimscore component to 127.0.0.1 (it is the suggested DNS configuration coming with openimscore documentation).

Is this configuration correct to trying register and messaging from machine B towards machine A?
