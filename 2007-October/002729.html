Hi Allesandro,

in short: just obtain an IP address and then register to the IMS (ideally the access network used to obtain that IP should not matter, but sadly e.g. for cases as Early IMS authentication it does). So any WLAN network which hands you an IP will do (no matter if this process is done in the 3GPP specific version (with Wx) or over a public hotspot).

For Wx: well, we did not implement it (so far ;) but is there a good reason to see that transport layer authentication as part of the Open IMS Core since I understand that it is not related to the IMS layer authentication ?

Allesandro,
if you are doing 802.1x authentication, you are using  Wx interface between the AAA and the HSS to be able to attach supplicant to the 3GPP packet network. As far as I know, there is not Wx implemented in the FoHSS. Supplicant should be able to perform EAP-SIM or EAP-AKA based authentication and create IPSec SAs towards PDG. The  UE is not affected. Andrej
