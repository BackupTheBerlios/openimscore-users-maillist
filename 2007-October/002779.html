Hi Jingyi,

You can not access the local loopback interface of another machine, so
yes, you need to change the P-CSCF. Also, you would better change the
I-CSCF and S-CSCF to this address and also update your DNS entries to
the new IP address from 127.0.0.1.

On the client, why not use the same DNS? Then you won't have any issues.
Anyway, even if you don't, it is safe to use the real realm name and not
the IP as all the communication will be done through the P-CSCF that you
have configured as IP address.

-Dragos

> > > The HSS (FHoSS) contains the user database and speaks the Diameter
> > > protocol to talk to the I-CSCF and S-CSCF. The CSCF’s use the
> > > Diameter protocol to ask the HSS for user credentials.
> > >
> > >
> > >
> > > When the HSS and the CSCF’s start they immediately try to contact
> > > each other and if not all of them are started they will not be
> > > able to talk. The error above are most likely because you have
> > > only started the HSS and not the icscf or the scscf.
> > >
> > >
> > >
> > > The configuration files for Diameter for the FHoSS is normaly
> > > located at
> > >
> > > /opt/OpenIMSCore/FHoSS/deploy/DiameterPeerHSS.xml
> > >
> > >
> > >
> > > And for the CSCF’s:
> > >
> > > /opt/OpenIMSCore/icscf.xml
> > >
> > > /opt/OpenIMSCore/scscf.xml
> > >
> > >
> > >
> > > These files contain information about what IP address and port
> > > number to listen on and what address and port the peers have.
> > >
> > >
> > >
> > > You don’t need to have httpd running. The FHoSS containts Tomcat
> > > and its own web server.
> > >
> > >
> > >
> > > Are you using httpd on port 8080?
