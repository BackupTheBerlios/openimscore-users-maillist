> How do you solve the problem with having a preconfigured image and DNS. Do
> you use DHCP for VMware? Does each image contain their own DNS? Would you
> like to explain how you do?

I don't have DHCP in VMware. There are two working scenarios. 

The first one uses the isolated "Host only" mode in VMware networking. This
works immediately if you configure your VMnet1 interface correctly.

The second (more comfortable) uses "bridged mode" and receives an IP address
from an external DHCP server (to be precise: I have it configured
statically). In this mode both host and guest have access to the internet.
This mode requires some adaptations in networking due to the local
environment.

I use a DNS server inside of VMware guest. It is pre-configured for three
test networks. So it is easy to e.g start a second image and build up a four
network roaming scenario.

> Do you have a build system to create the image and to keep it up to date?
No. I use the "configurator.sh" to make the necessary adaptations. It takes
me in total about 20 min to update OpenIMScore (a fresh compilation),
because I have done it very often.

> Does it only contain OpenIMSCore or other IMS related components?

Yes. I have also inside:
- SER + SEMS
- Openser
- SIPp

I have attached the README file (How to use the VMware image and what is
inside) to show the support I give to the students.
There is also an "install.log" file where all steps of building the system
are documented.
