> Hello Jin,
> 
> If you follow the same process, you will have at the end a bunch of
> machines which can run any of the components. What you actually start
> and how you configure them is up to you. The single box & 
> localhost-only
> guide is so just to minimize the time and pain of the new user.
> 
> Now, you should take a look at the *.cfg and *.xml files as well 
> as the
> icscf database (this contains network topology information) and 
> the DNS
> configuration of course. Well, just see what can be configured 
> from there.
> 
> If you have a multi-machine and/or multi-domain setup, here are some
> helping points:
> - you probably need DNS running on just one machine - multiple 
domains
> can be defined and on the others just modify the /etc/resolv.conf 
> to use
> the DNS from the other machine
> - for running just the HSS - only the Java related requirements 
> are a must
> - for running the CSCFs - only the ser_ims related requirement are 
> a must
> - MySQL db server is a must for both cases
> - if you have multiple HSSs, I would recommend that you give to each
> instance a different db (not a different DB server necessarily, 
> just a
> different database as in grouping of tables).
> - once you have the ser_ims installed on one machine, you can start
> there as many P, I or S-CSCFs as you like - just provide separate
> cfg/xml configuration files for each. What makes a particular CSCF 
out
> of the ser_ims is that cfg file that loads certain modules and 
enables
> certain message processing.
> - keep in mind that the cfg files for the CSCFs are just a 
recommended
> start point. You can do much more than the provided example.
> 
> Hope this helps...
> 
> -Dragos
