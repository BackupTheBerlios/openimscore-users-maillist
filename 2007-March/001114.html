The cscf_get_transaction log is not an error.

It is important to keep the I-CSCF on 5060 because the DNS for the home
domain would point there.

Then I don't think that the Open IMS Core was generating the 513
actually because in th 513 I see: Server: OpenSer (1.1.0-notls
(i386/linux)). Unless you have converted the Open IMS Core to also run
on OpenSer (which would be great, btw), I think that somebody else is
sending this and it probably does because of a loop (via count is 17
when this answer is generated).

So I think that actually you are not capturing all the traffic that goes
inside the Open IMS Core, as clearly the P-CSCF is not the one
generating that response.

-Dragos
