Hello Vignesh,

DSDOMAIN is regex match for slashed domain names mostly
in cfg files.You can see an example in scscf.cfg like "open-ims\.test".
These have to be substituted through the new domain name -with slashes of course, 
that's why it stands for.... 
open-ims\.test => new\.domain\.org

And DDOMAIN represent the match for "open-ims.test". After substitution with DDOMAIN:
open-ims.test => new.domain.org 

Frankly you don't need to worry about both. A simple execution would be
enough to re-configure your files ;) . Like Franz says, if you want to configure 
the domain for the second time then it is essential to change these(DDOMAIN DSDOMAIN) global variables.
