If you set the registration expiration to 60 seconds, after 60 seconds
you have de-registration, not re-registration. A trace would should show
that the SAR sent to the HSS is for registration expiration.

I don't know how the UCT client behaves, but the specs say that
re-registration should occur 600 seconds before expiration or 1/2 the
expiration period if expiration<1200 (or something like this). So
re-registration should occur 30 seconds before the expiration.

-Dragos
