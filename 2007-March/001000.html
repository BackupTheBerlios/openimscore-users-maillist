Hi all,

We feel that we should drop the SIP2IMS gateway from the project for
now. The reasons are:
- it creates more troubles than benefits
- the core now supports also MD5 authentication and as such its
functionality is covered

The sip2ims module will be deleted soon. If somebody objects and wants
to take over the development, we can create a branch for it, but until
we would have a good and working SIP2IMS Gateway, we won't have it in
the trunk.


Cheers,
-Dragos
