Hi Rishin,

TEL URIs can be used as public identities in a limited way, inside the
home network. To fully support them we need to modify SER's core to
support it.

The work would be more related to integration and verification of this
into the tm, sl, etc modules that do not have yet support for it. I say
integration because we already have an enum module capable of resolving
such URIs.

Our resources at FOKUS are limited for this task but we are offering our
full support if anyone else would want to try and solve it.

-Dragos

> Hi Babu,
>
> The Tel URI is not yet supported in FHoSS. We are working on introducing
> it, with many more improvements and fixes. However, in the interface, an
> IMPU should be using only one scheme at a time: either SIP, or TEL. I
> know that now the interface require the SIP as mandatory and is not
> using at all the TEL URI if is introduced. This is the explanation that
> is not working in your tests. This feature wasn't so important at the
> time of implementing and that is why remained not finished. However,
> this issue will be solved very soon and announcements about implementing
> it appeared already on Berlios.
