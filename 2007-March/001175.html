The roaming problem when using the UCT IMS Client only occurs when you
are using THIG and as such you have multiple Service-Route headers.
Simple solution - don't use THIG and use a scscf.cfg file from at least
r194.

Judging by the error:

ValidationException: The following exception occured while validating field: _serviceProfileList of class: de.fhg.fokus.cx.datatypes.IMSSubscription: Cannot validate a null Object.;
   - location of error: XPATH: IMSSubscription

I think that you forgot to attach a service profile to the user or manually provisioned the database or something like that. It looks like the HSS is unable to create the XML with the service profile for the given user.
