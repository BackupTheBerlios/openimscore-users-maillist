Hi,
I have seen this error as I installed OSIMS on
Solaris. I traced it how xcscf.cfg gets
compiled/encoded/parsed when SER starts. In my case
msg:len >=  max_len was always getting converted to
msg:len >= 0 hence I used to get this message
irrespective of the real message size. I had asked for
help in seruser mailing list but did not get a
response. As per Dragos's suggestion I commented out
that check in sip2ims/p/s/icscf.cfg files. 

Regards,
Sudhakar.
