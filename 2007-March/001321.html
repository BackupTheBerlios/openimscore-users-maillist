Hi Andrey,

Well, SER/OpenSER are not application servers. They are mainly SIP
proxies. I have suggested them for application server because it is the
easiest way from my point of view to do some simple applications. But
not necessary the best way.

You can start from scratch from SIP/Diameter stacks or you could use
some application server platforms. FOKUS is developing such a platform,
a SIP Servlet Execution Environment called SIPSEE, but that is not
available as open source.

In the end, as I said before, the OpenIMSCore should work with any AS so
it's not a question of preferring one. If you find some that we don't
inter-operate with and you think that it's our fault, we would certainly
appreciate some traces demonstrating the issue.

-Dragos
