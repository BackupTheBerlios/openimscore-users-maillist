> The Route: <sip:term at pcscf...> has the value saved
> during the
> registration in the Path header, by the same P-CSCF
> that it refers to.
> 
> This is used by the terminating S-CSCF to route the
> call towards the
> terminating contact, when resolving the public
> identity into the real
> contact address. The terminating IP address is
> usually in a remote
> network and the Path header indicates the way
> towards that. The P-CSCF
> recognizes it and identifies the case of Initial
> Request - Termination.
> 
> Getting back to the previous e-mail about SIP
> interfaces :-D , this is
> actually RFC3327, if you would like to know more.
> 
> As a difference, the mo (mobile originating) and mt
> (mobile terminating)
> markings are used inside dialogs, for subsequent
> requests. As the
> endpoints have already exchanged contact addresses
> and the routers on
> the way had the opportunity to record-route,
> subsequent requests no
> longer need to use the S-CSCF registrar or the Path
> information.
> 
> BTW: Service-Route (RFC3608 :-P ) is used for
> routing and identification
> of Initial Request - Origination.
> 
> -Dragos
