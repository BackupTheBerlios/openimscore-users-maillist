Hi Phuong Anh,

The difference is in signaling between IMS components. In the first
case, using iFCs will determine the originating S-CSCF to forward your
request to the Application Server.

In the second case, using PSIs, this forward will be done by the HSS in
the response to the LIR request. (The HSS will send the Application
Server Address in response.)
Will be something like this: UE->P-CSCF->S-CSCF->I-CSCF->AS
The I-CSCF uses the LIR/LIA mechanism to find out the location of the
destination user. As the destination user is a PSI, the response will
contain the AS address.

A very good pointer to this is: 5.4.12.2 from TS 23.228.

I have just made a fix for PSIs, as we had some mistakes in responding
for the LIR request with the AS address.

Cheers,
Adrian
