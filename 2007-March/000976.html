Hi all,

I have just merged in r161 the 01_persistency branch. What it does is
that the whole state on the P-CSCF and S-CSCF is saved into files or db
(db support only for S-CSCF for now) on a time basis and on shutdown.
Then whenever you restart the components the state is reloaded.

By default, the configuration files have this feature disabled and there
is a commented example for files too. Keep in mind that you need to
create a directory to store the files or a database to store the snapshots.

*The installation procedure has changed!!! *Even if you don't use this
new feature, you need to do an additional step before compiling ser_ims:

    * * New!!! * Do "make -f Makefile.ser install" in ser_ims/lib

      cd ser_ims/lib
      make -f Makefile.ser install
      cd ../..

We hope this helps and we will be very interested in feedback.

Thanx go to Mario Ferreira for contributing the db functionality which
also does incremental snapshots.

The old 01_persistency branch is now deleted in r162 and a new branch
for continuing work is now there - 02_persistency (branched from trunk).


-Dragos
