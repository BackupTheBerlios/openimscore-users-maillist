Hi Dragos

I modified icscf.sql. In the nds_trusted_domains, I have added
open-ims.com whereas I am using velankani.com for my users.
INSERT INTO `nds_trusted_domains` VALUES (1,'open-ims.com');

I debugged the issue. In I_NDS_is_trusted, it is picking subdomain from
via field and comparing it against value in nds_trusted_domains. Now via
field in the message coming contains the IP address whereas the
nds_trusted_domains contains the domain name in form of string. So the
comparison will always fail even if correct domain is added in
nds_trusted_domains table.

I have one more query. This is about writing cfg files. 
In icscf.cfg, from route{} route(NDS) is called. If route(NDS) fails,
still the execution will move ahead and wont break there. What is the
proper way to do this?
