> situation differ, if NAT is used/not used in P-CSCF?

Why are Port numbers in Via- and Contact Header different?
That is unusual. What client configuration do you use?

And now I try to answer you question. In case of NAT the network node
detecting the NAT-situation (e.g. by comparing the IP address in UDP header
with IP address in the topmost Via header field) adds a received-parameter
(containing the IP-address where the request was received) to the Via header
field and also a rport-parameter (containing the port-number where the
request was received). This network node is in our case the P-CSCF. Both
parameters (received and rport) are then used to forward the response to the
correct NAT-ed address..
