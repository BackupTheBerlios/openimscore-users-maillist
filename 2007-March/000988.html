Hi Joy,

the FHoSS follows the approach to have one subscription to be assigned
to n IMS Private User Identities (IMPI) which itself may be assigned
with IMPUs. So to setup the user accounts go to the section USER
PROFILES and create 

     1. IMS Subscription - assign the IMS Subscription (IMSU) name (e.g.
        Charlie) and save it, then add a IMPI to that IMSU
        
     2. Private Identity - the IMPI needs a NAI formatted username (e.g.
        charlie at open-ims.test ), the selection of supported
        authentication schemes (choose MD5 in case that you want to omit
        using the SIP2IMS gateway for registration) and a password,
        assign a Charging Info Set and the allowed roaming networks for
        the IMPI (subject to change soon) and save the it
        
     3. Public Identity - the IMPU needs either sip URI formatted name
        (e.g. sip:charlie at spice.org) or optionally a tel URI formatted
        name plus the assignment to a service profile from the drop down
        box, save the user and return to
        
     4. assign the IMPU to an IMPI - within a recently created Private
        Identity you can now assign an IMPU by pressing the “+” sign and
        searching for not assigned IMPUs (using % brings up a list), add
        one of them to IMPI by pressing the “Add” button
        
     5. Optionally, you can also set the GBA User Security Settings
        (GUSS) in the Private Identity section by clicking on the
        “configure” button for GUSS

Hope this is explicit enough. Alternatively, there is also a script that
Sven Bornemann contributed, please see
http://lists.berlios.de/pipermail/openimscore-users/2007-February/000644.html

Regards,

Peter
