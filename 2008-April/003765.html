Hi Dragos,

After investigation of some TSs I am not sure:)
The initial Proxy-CSCF discovery is applied so that UE may initiate
communication towards the IMS. So generally to send the first unprotected
REGISTER request. Like You said it can be performed through DHCP or as a
part of the establishment of connectivity towards the IP-Connectivity Access
Network.
According to TS 24.229 UE and P-CSCF shall support the Path header extension
from RFC 3327 and if they do the P-CSCF shall insert a Path header in the
request including the SIP URI identifying the P-CSCF. In RFC 3327 when UA
receives response for REGISTER request it takes the contents of the Path
header set to construct the correct Route header set, the same way registrar
knows the Route set to UA.
In TS 24.229 similarly S-CSCF knows the route to UE but UE knows P-CSCF URI
learnt through the P-CSCF discovery procedures. For me this is a little bit
confusing. Don't You think?

Cheers
Tomasz
