Hi Jerome,

well, we tried automatic detection as you say, but it does not work always. You can use HSS-Selected in the scscf.cfg, but expect to see errors for some of the algorithms that require more information to be transmitted and as such need to know in advance what algorithm will be used. Sure, for AKA and simple MD5 it works, but for at least one of the ETSI or CableLabs MD5 ones it does not (can't tell right now by heart for which exactly). Then for Early-IMS/NASS-Bundled you need to know in advance if you want to do that in order to also send the required info in the MAR. It's also true that you can in theory derive from some headers info about the UE and the AN, but I would say that this will just complicate a confuse system. 

Then the many parallel specified ways to do the same things and no real Common-IMS out there yet, does not help either in making this easy...

Cheers,
-Dragos
