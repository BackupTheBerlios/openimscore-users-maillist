Hi Tomasz,

The client does not learn the P-CSCF from the Path header, but from a 
DHCPv6 query for example, on _network_ attachment (IP layer, not 
SIP/IMS!). Using the value from the Path header will result in errors as 
it contains an indicator for termination and you can't claim terminating 
when originating messages, right?

-Dragos
