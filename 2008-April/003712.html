We've implemented call forwarding on unregistered.
But there is a problem when originator cancels the call.

For example:
Alice calls user named 'Busy' ('Busy' is offline) and the Application
Server redirects call to Bob. So Alice continues her SIP dialog with
Bob thinking that the opponent is user 'Busy'. And when Alice press
the 'Cancel' button she create request Cancel (From: Alice; To: Busy).
OpenIMS receives the 'Cancel' request, finds out that the user 'Busy'
is offline (you can see 404 error on IMS console screen-shot), wait
about 120 seconds and sends the 'Cancel' to application server.
Although OpenIMS delivers other requests (ACK, BYE) to offline user
'Busy' in the same SIP dialog to AS immediately.
