Hi Tom,

very interesting :). I was thinking of using an iPhone for that just as 
in the old days we used to have SER running on PocketPC. It's mostly 
useless to have a core network running on a phone, except that is über 
cool for showing off the performance.

Could you be more specific about the error? It will help a lot if you 
could post a stack trace taken with gdb. For that make sure that a core 
is generated on the crash (see ulimit) and then launch "gdb 
/opt/OpenIMSCore/ser_ims/ser <core_file>". Then issue a "bt" command and 
post back the output.

Cheers,
