If the value of Event header in SUBSCRIBE request is 'reg', the UE receives NOTIFY response from S-CSCF with a basic XML content whatever the service profile is default_as or another added AS.But if this value modifies to 'presence', the situation I said will occur, pcscf repeatedly sends request and scscf crashes. I suppose that the presence server(Openser Presence Module) is not configured well, although it runs well. BTW, I act as alice to SUBSCRIBE to IMS Open Core using presence_as as service profile instead. The settings of presence_as service profile are as follows:apsvr_id=2, name=presence_as, address=sip:open-ims.test:5065, default_handling=0;the values of as_perm_list table in database are all zeros;ifc_id=2, trigpt_id=2, apsvr_id=2, ifc_name=presence_ifc;ifc_id=2, svp_id=2, priority=0;spt_id=4, trigpt_id=2, sip_header=Event, sip_header_content=.*presence.*, neg=0, groupid=1, type=5;spt_id=5, trigpt_id=2, session_case=0, neg=0, groupid=0, type=2;spt_id=6, trigpt_id=2, session_case=0, neg=0, groupid=0, type=2;svp_id=2, name=presence_sp;trigpt_id=2, name=presence_tp, cnf=1; Soory about wrong format of the trace file, Dragos.
