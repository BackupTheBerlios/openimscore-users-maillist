Hello Rajeswari,

1. I think that you want to use the sipxPBX as an application server.
Then all you need is that sipxPBX follows the SIP RFC routing rules.

2. I think that you  might confuse things a little as the OpenIMSCore is
not and Application Server.

Then from your 2 ACKs I assume that they were for a 200 OK. Well, I find
it quite interesting and I really don't see the point of the
Record-Route in there. Then if I look at the Route headers, there is
definitely something wrong with the Routing of the INVITE request as in
normal signaling there is no way that you should obtain such a Route set
(twice mobile terminating at the S-CSCF). Please post the whole network
trace from the dialogs.

-Dragos
