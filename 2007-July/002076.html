When the S-CSCF triggers an AS, the S-CSCF adds a Route header in the
request with the AS URI and the S-CSCF URI.
When forwarding the request back to the S-CSCF, the AS includes a Route
header containing only the S-CSCF URI as it was received. The AS removes its
own URI from the Route header.

João Filipe Plácido
