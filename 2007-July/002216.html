Kamil,

you need to provision in the HSS the Application Server with its real
SIP URI. In this case, at sip:127.0.0.1:5060 you have the I-CSCF running
and you can not even start an App Server. So please choose something
else, start the AS there and configured that into the HSS.

Also be aware that there will be no automatic connection between the
Open IMS Core and the AS! The way it works is that the S-CSCF will
filter and forwards incoming messages according to the provisioned
service profile for the respective user, in the HSS. So you need to
generate traffic that would match the respective initial filter criteria
and would be forwarded to the app server, in order to see anything.

-Dragos
