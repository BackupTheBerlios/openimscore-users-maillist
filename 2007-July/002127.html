Hi Dragos

My Softphone( eyebeam ) is behind the NAT and is using STUN server to detect
the public IP (assigned by NAT) Which  it can  use in SDP for RTP
transmission.

EyeBeam-->NAT-->STUN

Eyebeam gets registered with public IP in the contact (  STUN server tells
EyeBeam its public IP)

So in INVITE message contact ip( public ) is different from via header
( private ip  ).And thus P-CSCF rejects it.

> The P-CSCF needs to identify the user based on the source IP:port. If it
> would identify it by contact, then anybody that finds out your contact
> (for example by calling you) can then impersonate your user  and make
