Sireesha,
 
> I am simply forwarding the INVITE message from AS to SCSCF.. ( The same
> INVITE message that I got from SCSCF earlier).
> Do I need to modify anything in the message at AS, before forwarding it to
> SCSCF? such as removing AS's own address from Route header..something like
> that?

Yes and No. 

No - if you have an AS behaving correctly as a compliant SIP proxy server.
In this case it will honour the Route header and forward the request
correctly back to the S-CSCF.

Yes - if you design your AS yourself. Then you have to modify some header
fields according to RFC 3261 for a proxy server. One of the modifications is
removing the Route header pointing to its own address as you already
mentioned. But there is also a Via header field ad eventually a Record-Route
header field which requires attention.
And first of all you have to define how the AS should behave: as a SIP proxy
or as a B2BUA. In the latter case something more has to be done.
