So those are examples of AS platforms, but the Open IMS Core project 
does not have the AS in scope for now, nor do we plan it at the moment. 
You can, of course, use any standards-conforming SIP Application Server, 
so you are not limited here by the core.

Then I have to mention that the CSCF base, SIP Express Router could 
potentially be used as an AS, although it is not designed and built with 
this target in mind, so it would be quite hard in the end.

Anyway, if anyone is interested in developing any other IMS components, 
we would happily accept them in our project.
