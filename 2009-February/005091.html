  Hi Sireesha,


  > I want to forward the "REFER" message to be sent to AS from SCSCF..


  I want to stress the fact that iFCs are only allowed to be used at "initial" SIP requests (see TS 23.218 6.2.9). 
  Initial SIP requests are dialog initiating requests or standalone requests (see the definition in TS 23.3218: Initial Request: a SIP request that either initiates the creation of a new dialog or is part of a standalone transaction.)

  REFER may be a standalone request or part of a dialog. In the latter case you are violating SIP standards when you modify the route during a dialog by setting a trigger on it.
  If you want to do some action on a REFER request within an INVITE initiated dialog you have to trigger on INVITE and pass-through alls other requests in your application except the REFER, where you act upon. This is because REFER within a dialog is not an initial SIP request.
