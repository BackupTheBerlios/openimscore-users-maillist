> Dragos Vingarzan wrote:
>> Hi Matsuoka,
>>
>> if I remember well, the trigger for IPv6 is the DNS resolution of the 
>> P-CSCF address. If you have an "AAAA" entry instead of an "A", the UE 
>> will try a IPv6 TCP connection to the P-CSCF. Upon connection, it 
>> would take the local-end address of the TCP connection as the 
>> outbound address towards the core. Then the TCP connection is closed, 
>> without sending anything and the SIP stack is bound to the local 
>> discovered IP address.
>>
>> So, this is the mechanism to determine the local outbound interface 
>> address that is able to rote packets towards the IMS core. Of course, 
>> several things must happen for a IPv6 use:
>> - have DNS entry correctly configured towards the P-CSCF, with an 
>> AAAA entry
>> - have the P-CSCF listen on TCP on that IPv6 address configured in 
>> the DNS.
>>
>>  From you log the connection failed. If you think that both points 
>> above were satisfied and still can't figure it out, please send a 
>> network trace.
>>
>> Cheers,
>> -Dragos
