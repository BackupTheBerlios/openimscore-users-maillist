Hi Vince,

> The purpose is to make some calls between IMS user (with IMS capable
> terminal) and VoIP user (with non-IMS capable terminal) wich are both
> registered on the same IMS core network.
> 
> I am just wondering if the VoIP terminal is capable to register to the IMS
> core (or if IMS needs so specific parameters in the register packet) ?

To be precise: you want to try a session between two IMS users with
different different clients (different capabilities).
Of course this is possible, because the P-CSCF is designed to be able to
recognize some missing header fields of pure SIP clients (e.g. Route header
field; see how "P_follows_service_routes" is handled in pcscf.cfg) and to
either reject or correct the request.
