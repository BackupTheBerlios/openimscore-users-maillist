Hi Vijay,

to distiguish the legs, you can in the FHoSS web console add to your
trigger point for that iFC the "session case" SPT information. TS 29.228
states:

Session Case class represents an enumerated type, with possible values
"Originating", "Terminating_Registered", "Terminating_Unregistered",
"Originating_Unregistered" indicating whether the filter should be used
by the S-CSCF handling the Originating, Terminating for a registered end
user, Terminating for an unregistered end user, or Originating for an
unregistered end user services.

Regards,

Peter
