Haven't tested with TLS (the sec-agree should be optional and not required).

here is my configuration though:

Service-Profile: IETF (works also with 3GPP, but I couldn't get the AKA
params out of my SIM card)
Public User name: sip:dragos at open-ims.test
Use Compression: No
Registration: Always on
Use Security: No
Proxy server:
    Address: sip:1.2.3.4
    Realm: open-ims.test
    User name: dragos at open-ims.test
    Password: ***
    Allow loose routing: yes
    Transport type: Auto
    Port: 4060
Registrar server:
    Address: sip:open-ims.test
    Realm: open-ims.test
    User name: dragos at open-ims.org
    Password: ***
    Transport type: Auto
    Port: 5060


Then just make sure that you have the user provisioned with MD5 in the
Open IMS Core.

Oh, and make sure that your "other" client will use the Allow headers
properly or else the N95 seems to be quiet.

-Dragos
