Hello Divya,

Well, we only tried it with 2.6. Anyway, the OpenIMSCore does not have
anything specific to the IPSec implementation. The interface is done by
calling simple bash scripts that contain the commands to set-up/remove
the SA. Please take a look at ser_ims/modules/pcscf/*.sh and feel free
to modify them if they are not right for 2.4+openswan. If you are
successful, we would appreciate if you would share with us the
experience and even write a small FAQ about it.

-Dragos

> Hello,
>
> If you see the second REGISTER in plain text and you are not running the
> client and the P-CSCF on the same machine, there, for sure, the client
> failed to set the IPSEC tunnel.
>
> The P-CSCF made the IPSEC associations and now is waiting for a message
> over an IPSEC tunnel. This is way it doesn't react to the second
> REGISTER.
>
> Florin.
