Open IMS is inserting the tag (d=0 or d=1) indicating Originating or Terminating leg in the second route header. This should ideally be the first route header that is sent to the Application server because that is the only header that is acted upon by the App server. All the subsequent route headers are just copied back in the message (Invite, Options etc) as is. 

I am attaching the trace showing this behavior. In the trace the following IPs are used:

10.72.0.60: open ims server
10.72.0.201: app server
10.3.0.29: Handset 1
10.3.0.42: Handset 2

Check frame 2 or frame 21 going from Open IMS to App Server in which the d=0 tag is inserted in the second route header instead of the first one.

Please let me know if my interpretations are correct and if so, can this can be fixed.
