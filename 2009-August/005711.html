Hi Mos,
 
> I just have a question about a case where an AS needs to send a SIP
> message to a registered IMS user.  The AS and has been provisioned in the
> HSS and a user has been attached to a service profile with the Term
> Registered session case pointing to the AS. However, when the AS sends the
> message, the S-CSCF responds with 403 Forbidden - Dialog not found on S-
> CSCF or Terminating user not suitable for unregistered services error. How
> do I need to configure my iFC etc; to allow the AS to initiate a session
> with an IMS user?

If you only want to send a SIP request originated from the AS to a
registered IMS user you do not need an iFC. The iFC controls the conditions
when a request is routed to an AS.

In your case your AS simply should send the request to an I-CSCF. The I-CSCF
then will find the S-CSCF where the user is registered.
