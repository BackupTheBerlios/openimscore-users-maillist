Hi Usman,

> 1. we are trying to register a user from some other IMScore system (XYZ)
with OpenIMSCore.
    This is a bad idea. A subscriber is registered in an IMS system. If it
is registered in another IMS system it does not need to register again (e.g.
if it is roaming).
    Why do you want to do this? What is the use case behind?

> 2. the moment we send the message from other domain (system xyz) to
OpenIMS the SIP message doesn't contain any authorization in the SIP header
message.
   Which message, REGISTER? 
   Where do you send a REGISTER request? 
   It only can be sent to a P-CSCF.

> 3. as a result it shows us error and didn't register
   ... as expected.
