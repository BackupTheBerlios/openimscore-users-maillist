Hi!
I am trying to get into the www.openimscore.org website to better understand what it really contains/not contain so I apologize for the following very basic doubts.
In order to avoid possible misunderstandings, please note that I do not want to criticize the openimscore.org website, but just to understand "how it works".
Sorry for raising this kind of questions on the mailing list, but I did not find replies on the website (at least not yet).
Many thanks in advance for possible replies.

For each individual user, the HSS allows hssAdmin to select one or more (up to "all") IMS Authentication Schemes:

 *   Digest-AKAv1 (3GPP)
 *   Digest-AKAv2 (3GPP)
 *   Digest-MD5 (FOKUS)
 *   Digest (CableLabs)
 *   SIP Digest (3GPP)
 *   HTTP Digest (ETSI)
 *   Early-IMS (3GPP)
 *   NASS Bundled (ETSI),
However:


    *   Since several IMS Authentication Mechanisms listed above have been introduced in the 3GPP standard from Rel-8 onwards, whereas  http://163.162.61.160:8080/hss.web.console/index.jsp says FHoSS - The FOKUS Home Subscriber Server (Rel. 7), I assume that this Rel-7 is NOT intended to be the "3GPP Release" (for instance because NAS Bundled was not inlcuded in the IMS 3GPP Rel-7). Is my assumption correct?
    *   A part from Early-IMS and NASS Bundled, I cannot map the other IMS authentication schemes listed above with those specified within 3GPP TS 33.203 (and thanks to the "Common IMS" process, 3GPP tooks the whole responsibility of specifying IMS). For instance, among the n.8 authentication schemes listed above, which one is supposed to be the "IMS AKA" (I mean the very first one IMS authentication scheme, specified since 3GPP Rel-5)?
    *   Which is the difference between Digest-AKAv1 (3GPP) and Digest-AKAv2 (3GPP)?
    *   Which is the difference between SIP Digest (3GPP) and HTTP Digest (ETSI)? Since 3GPP TS 33.203 does not contain both options, I am wondering why they are both considered by the openimscore.org implementation.
    *   Beside some individual IMS authentication  schemes (i.e. IMS AKA, SIP Digest, NASS Bundled and Early IMS) TS 33.203 Rel-8 (see Annex "P") also specifies their "coexistance". I am wondering if the TS 33.203 "Annex P" is already taken into account by the openimscore.org implementation or not. Since hssAdmin is allowed to flag multiple IMS Authentication schemes, I would say that the "coexistance" is already implemented, but since hssAdmin is also allowed to flag IMS Authentication schemes that do not exist within TS 33.203 I would say that the "coexistance" is NOT implemented yet...

Sorry for being so confused.
Thanks.
Regards,
Mauro
