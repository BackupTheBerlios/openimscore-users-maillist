Hi everyone,

I have tried to use SIP Client registration in the Open IMC Core.
I found the signaling flow is :

1.  UA   REGISTER                             --->   Open IMS Core
2.  UA           <--- 401 (Auth Challenge 1)       Open IMS Core
3.  UA   REGISTER (Auth Response 1)  --->   Open IMS Core
4.  UA           <--- 401 (Auth Challenge 2)       Open IMS Core
5.  UA   REGISTER (Auth Response 2)  --->   Open IMS Core
6.  UA           <--- 200 OK                             Open IMS Core


I don't know why the SIP UA is needed the secondary challenge/response.
(I have tried the X-lite, and other SIP clients, all are the same)

In the IMS UE, it always do only one challenge/response.

Could anyone explain that?
Any responses will be appreciated.
