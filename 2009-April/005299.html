The core does not deal with the RTP stream, other than some simple 
forwarding,  but that only when you enable an RTP Proxy. So both sound 
and video are exchanged directly between peers and not an issue for the 
core.

Then for roaming - you can enable roaming by simply creating a 2nd 
P-CSCF which will fill another P-Visited-Network-ID (configurable from 
pcscf.cfg). Then check the Visited-Network-ID mappings for each user in 
the HSS and enjoy roaming by using the 2nd P-CSCF from clients.
