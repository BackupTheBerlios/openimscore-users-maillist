Hmm... AFAIR, the charging vectors are generated only on initial 
requests and not on subsequent ones. Is there a spec that says it should 
also be in there?

In case it is, some new functions should be developed to save this info 
on the dialog probably and retrieve it on subsequent requests. the 
*_add_p_charging_vector() functions now only generate new values and 
fill them in, without saving or retrieving.

So it's not configurable. Contributions are of course welcome.
