> hi Sarette,
> answer to your questions follows
>
> >With wireshark my SIP/SDP invite is normal.
> >Have you the same problem?
>
> I have never had such a problem but it is possible that the SIP INVITE was
> badly parsed or that there is a bug in some or many of the functions
> involved in the AAR procedure
>
> the error you got comes from file sdp_util.c line 152
>
> try to investigate what is wrong with your SIP INVITE  / SDP or with the
> software as there are several known bugs in the P-CSCF with Rx support
> wehn you discover please notify me so i can add your patch to the branch
>
>
> >Then I wants to know if PCSCF must send a diameter message to PCRF when
> >he
> >receives a SIP register? My PCSCF-qos sends only STR diameter message
> >when
> >the session is terminated.
>
> In current standards it is possible that the P-CSCF sends an AAR message on
> SIP register (to allow the PCRF to perform session binding etc.. ) but this
> is not supported nor tested.
>
> But you can give it a try by modifying the config file and the functions in
> policy_control.c, pcc.c and pcc_avp.c to support this
>
> the idea is you need to create a new P_AAR and PCC_AAR functions which also
> generates an AAR message on REGISTER (which has no SDP)
>
> For more details look at TS 29.213 Annex B  Section 1 (Release 8)
>
>
> good luck!
