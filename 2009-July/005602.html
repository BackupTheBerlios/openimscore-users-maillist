I have checked TS 29.228 and found that the implementation is o.k. 
TS 29.228 says: 
"If the SIP Method SPT contains value "REGISTER", and no RegistrationType is
given, or if the S-CSCF does not support the RegistrationType attribute, the
SIP Method SPT matches to all REGISTER messages."

According to that the following values are set:
- for RegistrationType "Reg" 		value "0"
- for RegistrationType "ReReg" 	value "1"  
- for RegistrationType "DeReg" 	value "2"  

If none of the checkboxes are set no RegistrationType element is used in the
profile and the trigger matches in all cases. So everything is fine and
compliant.
