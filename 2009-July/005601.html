hi franz,

according to the XSD file

<xs:complexType name="tSePoTriExtension">
		<xs:sequence>
			<xs:element name="RegistrationType" type="tRegistrationType" 
minOccurs="0" maxOccurs="2"/>
			<xs:element name="Extension" type="tExtension" minOccurs="0"/>
		</xs:sequence>
	</xs:complexType>

this means it could happen 0 1 or 2 times but not 3

I do not know if this is wrong or not because the XSD files come in the 
TS from 3GPP (TS29.228 as you know) i have  just had a look at it and in 
Rel8 it is still with MaxOccurs 2

so we have to think if makes sense to have a service point trigger in 
the Trigger Point, for me I think why not. I think the error is that the 
content can only have values 0 1 or 2 but it could appear 0 1 2 or 3 
times and maybe 3GPP made a mistake (why not, we all make them) ...


but anyway a temporary solution is to substitute the maxOccurs="2" with 
a maxOccurs="3"  then it will work
that is in the file   ser_ims/modules/scscf/CXDataType_Rel7.xsd  line 206
