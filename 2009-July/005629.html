Hi Dushyant,

it's hard to put a number on this as there are way to many variables 
involved like hardware, topology setup, traffic profile, call durations 
a.s.o.

You should expect thousands of transactions per second and many hunderds 
of cps. The dumped core is not necessarily an overload situation, but 
more likely you have uncovered a bug.

To test for performance you will need to disable logging, reconfigure 
the CSCFs with hash-table sizes aligned to the number of users used, 
provision many more users so that you won't do thousands of calls per 
second per user (which is unrealistic), reconfigure the CSCF to use much 
more memory, etc.

Then you will hit the HSS limitations, due to its Java platform. For 
performance tests we used at FOKUS a different HSS written in C, which 
is much faster, but does not provide all the features that FHoSS does, 
like an Sh interface or the web GUI. Unfortunately, this is not 
available as open source and anyway it kind of goes beyond the testbed 
targets of our project due to its performance.

Cheers,
-Dragos
