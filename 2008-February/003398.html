Hi,

I have the following service scenario.
SIP MESSAGE request is sent by UE to AS identified by PSI.
When AS receives request it of course generates 200 OK response and sends it
to UE.
After that AS sends SIP MESSAGE to UE which sent the initial MESSAGE.

The situation and error arises when MESSAGE (AS->UE) is sent from pcscf to
UE.
The flow is the following:

AS->ISCSF->SCSCF->PSCSF->....?UE

When I do the first run everything works fine (always) but at the same time
almost always there is some problem in the next trials
(usually in the second)
The message which should be sent from PCSCF to UE is never delivered by
pcscf
or UE (here OpenIC_Lite) doesn't generate 200 response to SIP MESSAGE
request.

This causes that there is timeout and ICSCF generates 600 Busy everywhere -
Forwarding to S-CSCF failed request to AS.
