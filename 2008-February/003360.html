Hi Davide,

We strongly discourage you from using the /etc/hosts file. If you have
already configured a DNS server, why don't you just use it? All you need
to do is change the /etc/resolv.conf file on all your systems to use the
new DNS, that contains in one place all the addresses that you need.

We believe that it is quite trivial with a little help from google to
set-up and use a DNS server. So we really do not offer any support for
using /etc/hosts

-Dragos

>     Hi Davide,
>
>     You register to your domain. That is snodo. The Request-URI is
>     probably
>     sip:snodo, which is correct.
>
>     Then, the DNS for your home domain should point towards your I-CSCF
>     because the I-CSCF is the entry point in the network. So change
>     your DNS
>     and point snodo towards icscf.snodo or directly to .91.
>
>     -Dragos
