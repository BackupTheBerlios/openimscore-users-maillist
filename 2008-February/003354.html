Hi Tomasz!

> What is the meaning of the parameters in the Route header put in the SIP
> request when it is sent to AS
> Route: <sip:asluke.ims.rd.touk.pl:6060;lr>,
> <sip:iscmark at scscf.ims.rd.touk.pl:6060;lr;s=1;h=1;d=0;a=7369703a746f756b40
> 696d732e72642e746f756b2e706c>
> particulary s=1;h=1;d=0
> are they somewhere standardized in 3gpp TS or are only meaningful in
> OpenIMS SCSCF??

Route header parameters are only meaningful to the node which is addressed
by the route. Other nodes do not need to understand these parameters. 
The above mentioned parameters are only meaningful for OpenIMSCore S-CSCF.

In modul mark.c of isc-subsystem you can find the definition of these
parameters as follows:

 * IMS Service Control - ISC Marking Procedures
 * 
 * The specs say that the S-CSCF should somehow mark the message on the ISC
so that
 * when it comes back the S-CSCF can identify it. This should also be fairly
transparent.
 * 
 * The mark is as following: 
 * \code
Route: <appserver_uri>,
<sip:ifcmark@[isc_my_uri];lr;s=xxx;h=xxx;d=xxx;a=xxx>
  \endcode  
 *
 *		- s - skip
 * 		- h - default handling
 * 		- d - direction  
 * 		- a - the user aor encoded in base16
