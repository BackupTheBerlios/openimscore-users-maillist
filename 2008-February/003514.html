Jean-Yves,

We have had some experience with this and we found that the database will only accept ASCII characters. We could not find a way to directly insert the key value in HEX (or binary).

This meant that our client could only have keys that could be represented by ASCII characters.

For example on the client we set k to 0x61616161616161616161616161616161

And in the database script

'aaaaaaaaaaaaaaaa'

I could see that this probably isn't practical if you using a real ISIM unless you can set/modify the k value.
I'd mentioned this to Dragos at the IMS workshop in Nov. We haven't reported this as a bug. Mostly because we weren't sure if this was a real problem or just our lack of mysql knowledge.

Hope this helps

Mark Evans
Sprint Advanced Technology Labs
