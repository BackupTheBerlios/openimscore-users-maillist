Hi Pinak,

Well, because we can :). We thought that it would be much better if we
would let the network admins change the behavior of their networks,
irregardless of what some standards indicate. In the past with SIP
Express Router this has been proven to be a key-point because there are
a lot of SIP equipments out there not conforming with the standards and
in the end the admins had to do a lot of workarounds.

In the end, we think it's better this way: we have implemented the
helping functionality in the source code and then we're giving you an
example that tries to resemble the some standards. From there, it is up
to you to figure out how you want your network to behave. Maybe you
don't need security, or you want it more CableLabs like, or ... But in
any case, you don't need to know the secrets of pointers ;-) to change
some little things.

Don't worry about performance, we do quite well even with these scripts.

-Dragos
