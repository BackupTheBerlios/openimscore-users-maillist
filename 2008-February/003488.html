Hi Rakesh,

I will try to clarify.

> Thats not my question. We have already configured two clients which speaks
> MSRP. But we need an AS to establish SIP session.

I understand that you want to include an MSRP relay into the session and
this inclusion should be triggered by an iFC. For normal setup of an MSRP
session between two clients you do not need an AS and an iFC. Right?

> I think (May be i am wrong) OpenIMS does not have an AS embeded in it. The
> default port for AS given in OpenIMS configuration is 5065. But the
> problem we are facing that whenever we are redirecting the INVITE request
> to that port, error messages comes displaying that destination port is not
> rechable.

You are right. OpenIMS does not have an AS embedded in it.
The default port for AS given in OpenIMS configuration is only an example.
This iFC cannot be used when you do not provide the AS at the address
indicated (the MSRP relay server in your case).

> Then we tried with some other AS. But again faced the same problem.

What did you try? Modifying the iFC or providing an AS?

> So I want to know whether any special configurations are required in
> OpenIMS to establish the SIP session through some third party AS.

First: provide the AS
Second: use an iFC to route requests to the AS
The second point alone obviously results in "destination not reachable".

> AS we are trying to establish MSRP session does anybody have any idea
> about any AS that supports Messagaing and compatible with OpenIMS.

Sorry. No.
To be precise: you are looking for an MSRP relay server.
