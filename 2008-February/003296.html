Hi Nuno,

That's great :). So, the C/Java Diameter Peer stacks just need to have
some extra code to open TCP sockets on IPv6 if such an address is
configured as acceptor. In java I think it simply works and there isn't
much to do except check it on IPv6 alone and also on a mix of IPv6 and
IPv4.

Then, unfortunately the xcscf  code has to be checked for stupid things
that I did, like looking for 3 dots in an IP address. Probably the pcscf
registrar would have to be extended a little. Then the NAT code is
probably have to be a problem and should be again verified such that it
won't crash the thing even if enabled over IPv6. Overall, I would just
take a try, identify-failure, fix methodology because I do hope that
we've been relatively smart and left it open for IPv6, but unfortunately
also lazy for doing IPv6 parsing and checking when IPv4 is so simple in
comparison.

Let me know if you would need any help from my side. I don't know if we
actually need full branches for this.... maybe the changes are small and
maybe we could even put them directly in the trunk if they don't break
IPv4...

Cheers,
-Dragos
