No, it doesn't. The "Failed finding avp" in the I-CSCF is just a
warning. We do not have profiles for the diameter messages that would
tell us if an AVP is optional or not and as such we preferred to be
verbose and print logs every time some optional AVP is missing. In this
case, the I-CSCF did not find capabilities and result code in the UAR
(probable there was a server name AVP and an Experimental Result Code
AVP). But the authentication vectors are downloaded by the S-CSCF in the
MAR/MAA.
