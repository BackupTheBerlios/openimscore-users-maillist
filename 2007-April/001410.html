Hi Dragos,
Whatever you have said is true for TCP,
But When Client is using UPD as transport protocol, It should send Request from port-uc and expect responses at port-us.

Below is the excepts from 3GPP 33.203:
....
These ports are used with both, UDP and TCP. The use of these ports may differ for TCP and UDP, as follows:
        UDP case: the UE receives requests and responses protected with ESP on the port port_us (the"protected server port"). The UE sends requests and responses protected with ESP on the port port_uc (the "protected client port").
        TCP case: the UE, if it does not have a TCP connection towards the P CSCF yet, shall set up a TCP connection to the port port_ps of the P CSCF before sending a request to it.
....

Hello Prateek,

The responses are sent to the port_uc - User Client. When the Endpoint is doing UAC, then it sends requests from port_uc and should be ready to receive answers for those requests on the same port. Only requests are sent towards port_us.

Then about your error, please register the same contacts that you are going to use: you registered 192.168.1.2:5060 and then you use
192.168.1.2:1052 because of IPSec. So the error below was normal.

-Dragos
