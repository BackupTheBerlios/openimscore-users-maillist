Hi



i would like to check with you the following and know if it will work on OSIMS:

let's talk about a case where more than one IPMUs (let's say 2:IMPU1 and IMPU2) are associated to one IMPI,

(none of IMPUs are part of implicitly registered set neither concerned by wildcarded relation)



#1 IMS client register first IMPU1 successfully with associated IMPI in REGISTER request, meaning trigering authentication

and SAs setup between client and P-CSCF, that's OK



#2 after some time, IMS client wants to register IMPU2. My understanding is that it shall build a new REGISTER request for IMPU2

and send it using IPSec tunnel set up in #1.

In this case, whatever the authorization field in this 2nd register, the S-CSCF shall not trigg AKA because S-CSCF can validate that

the IMPI associated to IMPU2 is already authenticated (done with IMPU1 registration).

afterwards IMS client will be able to re-register / de-register IMPU2 independantly from IMPU1



> am I right? is it the way OSIMS works? (not able to test it for the moment)
