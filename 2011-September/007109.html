> After changing my ip addres in vmware using configurator.sh, I attempt to
> register using Boghe clilent, monser client from windows

>From your trace I cannot see what P-CSCF is really doing when it receives
the REGISTER.
I recommend you should use tcpdump for tracing on the IMS	platform. 
I use in this case the command "tcpdump -i any -s 3000 -w
/var/tmp/trace.pcap not port 22 and not port 3306".
(This command does not trace the SSH and MySQL traffic which is not required
for analysis).

In addition a log-file from P-CSCF might also be helpful if it turns out
that P-CSCF is the source of the problem.
