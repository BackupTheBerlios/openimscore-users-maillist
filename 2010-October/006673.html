Hi,

> I found out that this error appears when I use RTPProxy. When I set the
> rtpproxy_enable pcscf module parameter to 0 everything works.

So you already have an idea where the problem is rooted.
I propose that you should check the signaling (and what rtpproxy does) to
understand why P-CSCF gets the errors .

I also used the RTPproxy for the University-platform in Internet, and thers
I started rtpproxy with:
	/usr/local/bin/rtpproxy -l 195.245.225.190 -s
udp:195.245.225.190:34999 -u sip

and the parameters in pcscf.cfg were:
	modparam("pcscf","NAT_enable", 1)
	modparam("pcscf","ping", 1)
	modparam("pcscf","ping_all", 0)
	modparam("pcscf","nat_detection_type", 0x17)
	modparam("pcscf","rtpproxy_socket", "udp:195.245.225.190:34999")
	modparam("pcscf","rtpproxy_enable", 1)
	modparam("pcscf","rtpproxy_disable_tout", 60)
	modparam("pcscf","rtpproxy_retr", 5)
	modparam("pcscf","rtpproxy_tout", 1)

Maybe that helps.

ranz
