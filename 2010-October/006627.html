AFAIK the IMS core should eliminate information about that specific binding
(IP, port, etc) and that would make user status change from registered to
unregistered. I don´t know details about OpenIMSCore implementation, though.

About the notification of user registration status: I tried to make it work
on OpenIMSCore a few weeks ago, because it would be really useful to me to
know when a registration expires, but I found out that it it not yet
implemented and the work around I tried  did not either.
Maybe it is on the roadmap to implemente it in the future.

You can find more info here: http://www.faqs.org/rfcs/rfc3680.html
