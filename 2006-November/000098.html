Hi Suty,

The secret key in the HSS must be the same as the password in the 
SIP2IMS. Then HA1b is the one generated using gen_ha1 from private 
identity, realm and secret key, as following:

cartman gen_ha1 # ./gen_ha1 alice at open-ims.test open-ims.test alice
a3da98c9b36dde1f17e3f713225dc4d1

HA1 is not used and you don't need to worry about it. For some strange 
reason (as I said before) the HA1 generated by my internal PHP trials is 
different from the one generated by gen_ha1... still investigating this, 
but anyway, as not used, no worries.

I will add an entry to the FAQ with this.

-Dragos
