I am not sure what you mean by deploy.
The number of SER processes can be controlled from the *.cfg files in 
the children field and in the *.xml files in the workers field.
>  2)    and in place of "FQDN" and "REALM" do I need to change my 
> IP-address so that all can                  communicate properly.
>
no, that should be changed with your desired DNS names, that you should 
also change in the DNS.
you should change the IP addresses in the *.cfg files in the listen 
field and in the *.xml files in the acceptor element - where should the 
DiameterPeer listen on for connections
>          
>  3)    And What about SER ? Do i need to start it as normal action 
> (serctl start).?
we are providing the *.sh scripts that starts it and keeps it 
foreground, for debugging reasons.
>
>  4)    So , we need to add user in two places one is SER and another 
> in Hss (For end user who want         IMS)    is it right ?
>
No, only in the HSS. You need to add them also in the sip2ims database 
if you would use them through that gateway (same password and public-id 
must "sip:"+private-id!). If you use an IMS client, it is enough in the 
HSS, of course.
>
>   5)    And I am Using Mediaproxy as my mediaserver does it support 
> all types of media this IMS               protocols.?
>
the IMS core does not deal with any media, except when you would enable 
the RTPProxy - then obviously only the RTP media is proxied through 
there. I am not sure what do you mean here - the core components only 
deal with signaling.
>
