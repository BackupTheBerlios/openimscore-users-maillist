Hi Franz and congrats on being the first one to ask a question :)!

Problem 1: we clearly missed this. You are right, startup.sh is the 
script. We were supposed to rename that into FHoSS.sh and make by 
default that runnable from /opt/OpenIMSCore . We know also about the 
executable attribute and still have to convince ant to copy that too to 
the deployment location.

Problem 2: If you did "ant gen compile deploy" then there was probably 
some issue at compilation.
Please do a "ant cleanall". Then repeat the compilation and post the output.

Problem 3: No, the web interface is part of the HSS. The HSS contains 
and runs a tomcat instance. There are no prerequisites here, as such. So 
when you get it running, you should be able to access it at 
http://127.0.0.1:8080

I think that you might be one of the firsts to try it, besides obviously us.

-Dragos
