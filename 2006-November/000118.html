Hello,
capturing Diameter traffic between xCSCF and FoHSS, I found out, that Cx 
commands do not have Proxy bit set. According to the 3GPPspecifications  
29.229 and 29.329, each diameter command at Cx and Sh interface should 
have set P bit in the Diameter message header.

E.g.
See Wireshark traffic for UAx attached : Diameter Protocol -> Flags -> 
Proxyable
See 29.229 for UAR <Diameter Header: 300, REQ, PXY, 16777216>, UAA 
<Diameter Header: 300, PXY, 16777216>
