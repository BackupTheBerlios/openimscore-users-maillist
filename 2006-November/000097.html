Hi,

some details here:

Provisioning of User Profiles
The final step in user provisioning is to assign the created service
profiles to IMS Public User Identites (IMPU). The FHoSS follows the
approach to have one subscription to be assigned to n IMS Private User
Identities (IMPI) which itsself may be assigned with IMPUs. So to setup
the user accounts go to the section USER PROFILES and create

        IMS Subscription - assign the IMS Subscription (IMSU) name and
        save it, then add a IMPI to that IMSU
        
        Private Identity - the IMPI needs a NAI formatted username, the
        selection of supported authentication schemes and a password
        (secret key that ideally would be stored in the UICC but now has
        to be stored in the SIP2db as well ;), assign a Charging Info
        Set and the allowed roaming networks for the IMPI and save it
        
        Public Identity - the IMPU needs a mandatory sip URI formatted
        name (e.g. sip:user at open-ims.test) and optionally a tel URI
        formatted name plus the assignment to a service profile from the
        drop down box, save the user and return to assign the IMPU to an
        IMPI - within a recently created Private Identity you can now
        assign an IMPU by pressing the “+” sign and searching for not
        assigned IMPUs (using % brings up a list), add one of them to
        IMPI by pressing the “Add” button

These steps should allow you to provision the FHoSS with new users and
by modification of the specific service profiles, you may detail the
authorisation levels of user groups to specific services

Hope this solves the question.
