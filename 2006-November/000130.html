Hi, 

 

When establishing calls the Invite that reaches the P-cscf contains a Route header that looks like this: Route: <sip:pcscf.open-ims.test>,<sip:orig at scscf.open-ims.test>

 

But the service-route that P-CSCF stores during registration is the following SR : <sip:orig at scscf.open-ims.test>

 

So when the P-cscf receives this Invite and calls the P_follows_service_routes(), it will get false, which is wrong, since the Request follows the service Routes.

Since the P_enforce_service_routes is called if the P_follows_service_routes() fails, this will result on the request leaving with two Route headers with the following content : <sip:orig at scscf.open-ims.test>
