Hi all,

1. Lebing, I have no idea why you needed any C++ stuff when all our code 
is pure C. I am not using either ubuntu or debian, but I doubt that you 
would need libmysql++-dev as SER uses the C API for MySQL, usually 
called libmysql-dev (on Gentoo it is included in dev-db/mysql).... maybe 
you could enlighten us here...

2. uuid-dev - again no clue. I know all the code in the CSCF extensions 
and most of the SER's routines, but never encountered this. What is it 
and where did we use it??

3. MySQL - if you would use the provided database scripts, all the 
rights are set by default, so that you will not have issues with it. So 
far nobody complained. Maybe you are talking about an old internal 
version that you might have seen. Please update.

4. Domain and DNS related - again, almost all pre-configured, plus that 
this problems are off-topic here. A google search will solve them.

-Dragos

>>> You need the ipsec-tools package installed (at least this is 
>>> how it is called in Gentoo Linux). The setkey is a utility 
>>> that manipulates the Linux kernel 2.6 IPSec implementation. 
>>> You probably also need your kernel compiled with AH and ESP 
>>> for IPSec if you plan to actually use the IPSec tunnels (if 
>>> not, just comment the 2 lines in the pcscf.sh and forget about it).
>>>
