Hi Justino,

First, thank you for the ubuntu/debian clarification. So you only need 
libmysqlclient15-dev.

Regarding /etc/hosts - can you make it do reverse-dns?

Regarding the packaging in deb/rpm/ebuild: at the moment the targeted 
audience is the IMS developers and not the regular users. Packaging 
would make sense once we would get some releases out. However, as the 
current state is far from that, it would take some time. Until then, I 
think it would be for the best to only support the last SVN revision or 
else I don't think that we could cope with the 
development/testing/packaging effort.

IMO, the first objectives go towards:
- stabilizing it
- securing it
- compiling a list of feature requests.

So I would greatly appreciate your feedback on any of these while you 
play with it.

I know that for many this version is already fully usable, but I would 
just like to set the standards a little higher and tag a release only 
when we really have something good.

Cheers,
Dragos
