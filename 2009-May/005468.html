> Adam,
>
> as said below you need in principle only one statement "loose_route()" to
> see that a request is forwarded to OpenSIPS as an AS and sent back to the
> S-CSCF. But this does nothing more than demonstrate that the forwarding
> mechanism (iFC) to an AS works. This does not apply any feature.
>
> But when you want to implement a specific feature, that is another issue.
> Then you have to define which feature you want to implement and how to
> implement it with OpenSIPS. In this case you will soon recognize that
> OpenSIPS is not as flexible as an application server for IMS as perhaps
> expected. But in principle it can be done.
>
> I don't know if anyone has done such an exercise. I suggest it is more
> worth to spend some time in using a dedicated application server platform
> supporting converged services (SIP + HTTP).
>

>> Adam,
>>
>> in principle it is not difficult to connect OpenSIPS to OpenIMScore. You
>> just need one statement inside of OpenSIPS ("loose_route()") and it will
>> return the request perfectly to OpenIMSCore. What Dragos meant - and I agree
>> on that - is that OpenSIPS / Kamailio / SER etc... have not been designed to
>> be an application platform for IMS.
>>
>> Regarding a 3rd party REGISTER via an iFC you cannot authenticate REGISTER
>> again because the S-CSCF cannot answer with valid credentials.
>> And usually you do not need to authenticate again because there should be
>> a trust relationship between S-CSCF and AS.
>> Each REGISTER request forwarded by S-CSCF should be regarded as already
>> authenticated.
>>
>> -franz

>>> Hi Adam,
>>>
>>> OpenSIPS, just like OpenSER, SER, Kamailio, etc, are at the core the same
>>> thing. They are all forks of the original SER project, which we also use as
>>> a base for our CSCFs.
>>>
>>> Then the answer to your question is the same as before. It is certainly
>>> possible to use a SER-based system as an AS and in many cases it was one of
>>> the best choices, but we would not recommend it as SER was not designed or
>>> implemented for serving as an AS platform, as understood in the NGN
>>> standards.
>>>
>>> Cheers,
