Evans, Mark [CTO] wrote:
>
> Hi
>
> I just quickly looked at your pcap file. Seems like the QOP value in 
> the Authorization response is set to Auth-int. Historically, this has 
> been a problem and is discussed in the OpenIMS archives. I don’t think 
> this has ever been fixed in the core.
>
> To force the client to use the default QOP of “auth” do the following 
> and try again.
>
> In the file scscf.cfg find and comment out the line
>
> modparam("scscf","registration_qop","auth,auth-int")
>
> And replace it with
>
> modparam("scscf","registration_qop","")
>
