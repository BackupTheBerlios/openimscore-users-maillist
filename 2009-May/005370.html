> So i have made a change at scscf.cfg so that openIMSCore routes to PSTN
> when called number is sip:123.
> Is it ok to do this? 

In principle yes, but that should not be necessary, because interconnect
towards SIP based networks should be possible.

> what should be the actual format of the land
> telephone number when it is sent to the PSTN gateway from openIMSCore?

This is usually a TEL-URI.
If you use a SIP-URI, it should be a in the format of a true E.164-number
(preceded by '+' and the international prefix and with a 'user=phone'
parameter added).
