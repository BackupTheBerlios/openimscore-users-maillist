Hi Chung,

failed calls when calling yourself unfortunately is a known and still
outstanding bug in the Open IMS Core. So at the moment you can only
work-around on this issue with a client that uses different IP:port
tupples for each line.

I used 2-4 port SIP equipments before, but for me it was never a hard
requirement to be able to call the same equipment back. So other than
this particular scenario, all went fine.

Cheers,
-Dragos
