Hi Tomas,

> finaly i figured out what was the issue about delivering SIP messages to
> application servers (PSIs). the first thing is, that i HAD TO manualy
change
> user_state of the PSI in IMPU table of mysql hss_db database  to
> REGISTERED. without the change the messages are not delivered.
> the other thing is, that i had to add Route: header field containing
> sip:mt at scscf1.domain1:6060 into messages sent to PSIs. i don't think that
> MT@ should be there as the PSI is not mobile terminating, it is PSI.
> i also had to override SCSCF and add a condition into scscf.cfg file which
adds
> correct port number to URI of the SIP messages. i think that the port
number
> should be read from DB by scscf and inserted to the URI (as the port
> identifies the AS also) but apparently openims does not do that.
> 
> finaly, i achieved 2 interconnected openims cores which are able to
transfer
> SIP messages from one application server in one core to the other AS in
the
> second core.

Thank's for sharing your experience.
