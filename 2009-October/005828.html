> Caller: sip:alice at open-ims.test
> Callee: tel:+30377710001 (Implicit IMPU of sip:bob at open-ims.test.Notify
> Request for RegEvent Subscription contains tel:+13037771001 as active)

The problem is now obvious: You have to add an ENUM record for each tel-URI
you are using within your network into DNS pointing to the sip-URI of the
subscriber. Otherwise - this is what happens in your case - the request is
forwarded to a PSTN gateway (default port 9060).

BTW: A more usable trace can be created with a tcpdump command like
   tcpdump -i any -s 3000 -w /var/tmp/trace1.pcap not port 3306
