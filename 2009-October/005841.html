> How can we have "Diversion" in an Invite request through OpenIMS? Is

That task has to be solved in an application server and it should be
implemented as specified in 3GPP TS 24.504  (PSTN/ISDN simulation services:
Communication Diversion).

That's a good exercise for understanding the attachment of an application
server via iFC.
