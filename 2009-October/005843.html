Hello Gowtham,

> I have setup a IMS core with cisco sip gateway as the PSTN route. I
> can place a call from the IMS client by using tel:TN. But when a call
> comes into the IMS core to the TN on the IMS core. The core responds
> to the gate with 404 user not found on S-CSCF. Any clue on what am i
> doing wrong? I can provide any trace.

I guess that you are not aware that IMS subscribers are only to be addressed
via sip-URIs. In case a tel-URI is used you have to provide an ENUM entry to
map it to the tel URI of the subscriber.
