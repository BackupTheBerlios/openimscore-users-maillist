> Just a few remarks:
> If the Digest-AKAv1 authentication and security mechanism is used only the
> REGISTER request is authenticated. After that a security association is
> setup and a further re-authentication is not necessary as long as the
> security association is active.
>
> If SIP Digest authentication is used without TLS then the S-CSCF may
> perform
> authentication for every request. This is the case where "407 Proxy
> Authentication required" is used.
>
> OpenIMSCore does not support the SIP Digest authentication for other
> requests then REGISTER. Therefore additional code in S-CCSCF is necessary
> to
> support that. The main difference is:
> In case of Digest-AKAv1 the S-CSCF gets authentication data from HSS when
> processing the REGISER request.
> In case of SIP Digest authentication for non REGISTER requests the S-CSCF
> has to calculate authentication data itself. Therefore it has to store the
> password which it got during registration and has to apply the SIP Digest
> authentication autonomously. This requires some code in S-CSCF, but it
> should not be that difficult and may be an interesting exercise.
> This cannot be done by doing some modification in scscf.cfg.
>
> For more details look into TS 24.229 chapter 5.4.3.6.
