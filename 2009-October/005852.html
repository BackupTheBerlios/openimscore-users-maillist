Hi Franz,

It's OK!
I have checked the TS 24.229 release 8 again, it notes the following:
5.1.2A1.1 "When SIP digest is in use, upon receiving a 407 (Proxy
Authentication Required) response to an initial request."

5.4.3.6.1 "If the request does not contain a Proxy-Authorization
header field or the Proxy-Authorization header field does not contain
a digest reponse, the S-CSCF shall:
     a) challenge the user by generating a 407 (Proxy Authentication
Required) for the received request..."

The conclusion is when SIP digest be used, the 407 response will be needed.
