Hi Arjun,

> I did a call test and took some traces, by looking into the traces i found
> that SCSCF sends multiple SUBSCRIBE requests, and there is a server error
> and a loop detected messages in the call flow, how ever my call set up was
> successful, Can some one help me to point out what is going wrong? or
> solving this problem?

You have to look at the content of the SUBSCRIBE request and you have to be
aware that you obviously do not have a central server to handle presence,
presence.winfo and message summary event.

If a SUBSCRIBE request is addressed to arjun at ... or alice at ... you should not
be surprised that it is really forwarded to arjun and alice if you do not
have any iFC activated which directs the request to e.g. presence server.
