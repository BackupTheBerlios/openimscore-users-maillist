Hi all,

on Monday I participated in a very interesting meeting:
http://sip-router.org/

The basic idea is that the main developers of the SER and OpenSER
projects have reached an agreement to work together towards a common
project. First, a unified core and tm will be realized. Then hopefully
all the modules will be made compatible to the common project.

Also, the core developers were quite open towards the Open IMS Core
project and we're hoping to include all of our remaining changes outside
of the pcscf/icscf/scscf/isc/cdp modules in the main SIP Router project.
Next we need to make sure that our modules work with the unified core.
Then the Open IMS Core will be transparently usable with the common
SER+OpenSER project and we will be able to benefit from all the new
features in SER and OpenSER.

Our project will of course go on as a separate community with the same
main scope of IMS experimentation. Some parts and base functionality,
like for example the Diameter module (cdp) will probably be adopted
quite soon outside of our project, but others, due to their unstable
nature will not. In the end, my hope is that all the source code
repositories will be joined and linked, such that you could easily
checkout the entire SER/OpenSER functionality, including the Open IMS
Core enablers.

So if you are interested in this effort unification, be sure to check
their website and let's discuss how we could contribute back with IMS
extensions!

Cheers,
-Dragos
