Hi Tran,

> I have used OpenIMSCore to connect my server AS. I have configured
> Trigger Point to forwards message : REGISTER, INVITE and MESSAGE to my
> server AS.
> But the OpenIMSCore has forwarded message INVITE and MESSAGE to my
> server AS, and message REGISTER only arrive to SCSCF, it isn't
> forwarded to my server AS.
> In Trigger Point, I have configure : Disjunctive Normal Format
> Add SPTs to Trigger Point  : SIP Method (REGISTER, INVITE, MESSAGE)  +
> Session Case (Origin Session)

I propose that you take a trace of registration procedure of all core
messages (SIP and Diameter) with tcpdump. Then look into the trace with
Wireshark and check the SAA request sent from HSS to S-CSCF during
registration (you will have to manually set "decode as diameter" when you
click at the packet). Check the User-Data AVP (Copy as printable Text ...).
This clearly shows the XML structure of the iFC. Then you will se if
REGISTER is included as expected.

If you need further help add the tracefile.
