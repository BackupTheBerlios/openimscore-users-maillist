I have NAT enabled on purpose, because the OpenIMSCore platform is in public
Internet and I also use clients behind NAT.
But in case of IMSdroid the client uses also a public IP and the NAT
mechanism should not change the port.
When I disable NAT-handling the SDP is not touched by P-CSCF (as expected)
but then my clients behind NAT won't work anymore.

For me it is clear the issue is somewhere in the code of module sdp_util.c
in pcscf. Whenever it touches the SDP to add the attribute
"a=nortpproxy:yes" it erroneously also changes the port, but I did not dig
into the code to find the bug.
