Hi,

> 1. What can be the possible reason for this one user getting registered
> but the other facing authentication failure issues?

The only explanation I have is: the secret key does not match.

> 2. Regarding viewing the Diameter interaction messages for any successful
> registration procedure , for example in this case when user Trang is
> getting registered , where can i view the Diameter interaction messages.
> Even though in wireshark amongst enabled protocols Diameter is present but
> i can't see any exchanged Diameter Interaction messages like
> (UAR/UAA/MAR/MAA). Where to see this exchanged messages between I/S-CSCF
> and HSS?

UAR/UAA/MAR/MAA is contained in packets 243, 282, 352 and 432.
You have to click on the packet and explicitly "help" Wireshark with "decode
as Diameter". Wireshark can not automatically recognize the packets as
diameter protocol because the default port is not used.

> 3. My last query is that i want to register both my users simultaneously
> one by one and not by de-registering one user and then registering
> another. For this thing to do i guess i need to have 2 different IMS
> enabled clients. For example , presently i have monster client where i am
> registering Trang. Now to register Jelena i need to de-register Trang and
> then register Jelena but i don't want to de-register Trang.

Take a 2nd client for this or a second computer if you want to use the same
SW-client. Mercuro is a good choice for another 2nd client.

Good luck.
