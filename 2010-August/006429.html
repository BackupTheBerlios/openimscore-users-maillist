> They do not contain the diameter messages between the cscfs.

Yes. For that you have to run e.g. tcpdump and logging all messages (maybe
selective) in a file.

> What I was looking for was Open IMS logs, more 
> importantly the messages which show up on stdout when we run pcscf, scscf
> and icscf . I started running them in the background and hence cannot see
> them. Is there a way to see them without restarting the core ?

Exactly these messages are part of syslog, at least in my case.
I guess you have set the "log_stderror" parameter in your xcscf.cfg-files to
"no". Then the following rule applies:

With the "log_stderror" parameter you can make OpenIMSCore to write log and
debug messages to standard error. Possible values are:
- "yes" - write the messages to standard error
- "no" - write the messages to syslog

If you have left "log_stderror=yes" unchanged, then maybe the message are in
/var/log/messages.
