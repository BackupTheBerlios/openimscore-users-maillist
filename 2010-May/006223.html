Hello Alex,

> Is it possible to disable multiple location registration on OpenIMSCore
> ?(, i.e., IMPI+IMPU can only register once even if the IP is different).
> Is there a feature where the later Register request would replace the
> previous entry on the registrar_storage hash table?

Such a feature is not in-line with IMS architecture.
There is a feature (notification in case of loss of signalling) which is
provided by Gx interface (TS 29.212). I understand that this notification
should be used by P-CSCF to de-register the client.
But such a feature is not implemented in OpenIMSCore.

> I'm facing a problem where the scenario requires that all INVITES be
> forwarded to an AS through an IFC. However, when an IMS Client (User
> Agent) crashes or losses connectivity and the same IMPI+IMPU registers
> again from another location (different IP and/or port) and then tries to
> send a INVITE, the PCSCF answers with a "403 Forbidden - Your must
> register first with a S-CSCF".

I cannot reproduce this behaviour.
If I simulate a crash of a client and re-register with a different
IP-address I get an additional registration which has to be deleted
manually.
