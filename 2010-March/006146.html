Hi All,

Here is what we can find on specification:
The Registration Status, specified in 3GPP TS 29.228, contains the status of
registration of a public user identity

   - Not Registered State: the user is not Registered and has no S-CSCF
   assigned.
   - Registered State: the user is Registered at the request of the user and
   has an S-CSCF assigned.
   - Unregistered State: the user is not Registered but has a serving S-CSCF
   assigned to execute unregistered state services.

On OpenIMS, what is the process telling that user should be Unregistered or
not-registered?
When a user de-register from openIMS, what should be its state?

why does user come from Unregistered to Non-registered mode doing nothing
(when checking mysql database)?
