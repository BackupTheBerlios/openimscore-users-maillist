Hi Ket,

> To go further about this subject, I'd like to know what is the De-
> registration behavior please:
> When a basic subscriber (with no IFC enabled) send a REGISTER to ims CORE
> with expire=0, does the state in HSS become Uneregistered or Not-
> Registered?

According to the standards both options are valid. This depends on the
S-CSCF and is reflected in the AVP parameter "Server-Assignment-Type" of the
SAR diameter message. The two options are:
If S-CSCF does not keep user profile stored:
- TIMEOUT_DEREGISTRATION (4)
- USER_DEREGISTRATION (5)
If S-CSCF keeps user profile stored:
- TIMEOUT_DEREGISTRATION_STORE_SERVER_NAME (6)
- USER_DEREGISTRATION_STORE_SERVER_NAME (7)

I found that there is an exported parameter of S-CSCF which might control
this behaviour "server_assignment_store_data" but had no chance to dig
deeper.
