Hi Vikas,

>From your output I'm guessing one of these is happening :

1) The Diameter library was not compiled and cannot be found on CLASSPATH

2) You checked out new source code over the new one, and for some reason
(permissions?) the new version of Diameter lib files wasn't created on disk

3) Your FHoSS startup file does not contain correct CLASSPATH definition, as
you probably know, you can modify your CLASSPATH in this file.

If all else fails, I would suggest checking out new source into a new
location.
