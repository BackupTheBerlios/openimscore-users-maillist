Hi Reza,

Not sure if I understand your questions exactly....

The client <-> IMS communication should always go through the core. So
the client should talk only with the P-CSCF, always.

In your case, the AS should probably create a B2B call by sending 2
INVITEs, one towards each client. All should be routed through the Open
IMS Core. Normal routing would do just fine.

In case of AS doing origination, you will need to also fill in a Route
header with something like: <sip:open-ims.test;orig>. For termination,
just forward the request to the I-CSCF.

Cheers,
-Dragos
