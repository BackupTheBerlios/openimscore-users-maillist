The thing that you probably missed is that the CSCFs, which are C based,
have both SIP and Diameter interfaces. So the CDiameterPeer is used as
the stack library in the CSCFs and the JavaDiameterPeer in the HSS. The
Cx interface is as such between the CSCFs and the HSS, between the 2
stacks. You can, of course have a C based HSS or a Java based CSCF, if
you would want to implement it like that.

So any of the 2 stacks can be used on any interface. The choice is only
on the programming language and not at all on the component type of
interface.
