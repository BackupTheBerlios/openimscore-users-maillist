Hi Antonio,

> I have only two doubts: if I like to achieve connect with softphone
> auntenticacion akav1 and softphone md5 I left unmarked in scscf.sh file
> both options?  

In this case you configure your S-CSCF (in scscf.cfg) for " Let the HSS
decide":
  modparam("scscf","registration_default_algorithm","HSS-Selected")
and set in the HSS (Private User Identity) the authentication method you
want to be used.

> ... and the error messages exposed in the other email are normal? What can
be the reason for this messages? configuration maybe?

The warning
  WARNING:peer_connect(): Error opening connection to to ... port 3868
connection refused
is normal as long as it is not permanent. It is caused by the fact that you
cannot start HSS and S-CSCF/I-CSCF in parallel. The one which is started
earlier cannot find a responsive peer as long as the peer is not started.
