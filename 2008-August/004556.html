I have extracted the user-data transferred from HSS to S-CSCF during
registration (SAA command, packet 456, see attached user-data.xml). From
that you can see:

You have 3 trigger points active
- Prio 0: (method = SUBSCRIBE OR PUBLISH) AND (Event header content =
.*presence.*)
- Prio 1:  (To header content = .*skype.open-ims.test.*) 
- Prio 2:  (To header content = .*iptv.open-ims.test.*) OR (To header
content = .*media.open-ims.test.*)

According to the INVITE request containing "To:
<sip:bob at skype.open-ims.test>" trigger point with prio 1 matches as you
expected. The application server responsible for this trigger point is
contrary to your expectation 192.168.3.6:5060 (see line 91 of
user-data.xml).

If this is the wrong application server you have to modify the address of
the application server accordingly.
