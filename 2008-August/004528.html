Hi Miroslave,

1) yes. Fill the auth id of the Sh interface.
        <Auth id="16777217" vendor="10415"/><!-- 3GPP Sh -->
If the 2 Diameters peers won't have anything in common, they will not
establish a successful connection. Also messages are not routed towards
peers that do not support the specific application ID that they are for.

Also fill in the HSS as a Peer.

2) No. you only need to change the HSS Diameter configuration and
include the Sh client as a Peer (if the HSS has the AcceptUnknownPeers
configuration item set to 1, then you don't necessarily need to
configure it here, but then the connection attempts will happen only
from the client's side towards the HSS - which is not critical, but it
might take a little longer until you have an established and usable
connection). Then remember to configure the Application Server rights on
the Sh interface, from the HSS web interface.

3) Here you have the HSS itself as example on how to create, send and
receive Diameter messages. diameterpeertest.java is just a simple use
example.

Cheers,
-Dragos
