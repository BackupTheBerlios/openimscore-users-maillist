Hi,

 

Seems like you’re a little confused with the C & J prefixes. They refer to
the implementation language, not the interface (Sh, Dx, Cx,  etc.)!

 

Both are Diameter-stacks. So it is up to you to choose which one to use,
depending on your chosen implementation language. So, if you want to use
Java then go for JdiameterPeer, and if you want to use C then go for
CDiameterPeer.

 

Since you have a C-library that you want to use, then it might be better to
use the CDiameterPeer.

 

But if you really prefer to work with Java and will have to do a lot of
other implementation work on top of the mentioned C-library, then it is
possible to do so too. This is called Java Native Interface (JNI) and you
can find information about that at http://java.sun.com/docs/books/jni/.
