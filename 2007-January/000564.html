If I may detail a little... if you just don't care, ignore it ;-)

I have enabled by default NAT so that we would get rid of a lot of
complaints about it. Anyway, 2 bytes sent even if not required is not
that much traffic. The sequence sent, CRLF, is small and should be
easily ignored by any SIP stack. It is used to keep the NAT pinholes
alive on the NAT boxes. It is quite a nice trick to eliminate nasty NAT
problems.

-Dragos

> This is caused by NAT handling in P-CSCF which is enabled by default
> (NAT-pings). You can disable it by setting the "NAT_enable" parameter in
> pcscf.cfg to zero.
