I wanted to emphasize that a public identity is a sip or a tel uri, not
both.

About PublicIdentifiers you are right, it contains more
IMSPublicIdentities, so is possible to have more tel URI-public
identities and more sip URI-public identities for the same user.

But, our interface has a bug, and does not permit to add a tel URI
- -public identity. The SIP URI is mandatory in the interface, so only SIP
URI-public identities can be added  :(  Even that you saw the filed for
tel URI, this make no sense, as always SIP URI counts. It was a
misunderstanding when was implemented. It should be the possibility to
chose SIP or TEL.

I hope that I am more clear now  :)

- -Adrian
