> Yes we did. We have here at FOKUS many app servers. Some, like our
> presence AS, are built using NIST-SIP or other stacks. We also have
> developed SIP Server Execution Environment (SIPSEE) and we are
> successfully using that too. Overall, it's not a big deal: you can proxy
> messages from the S-CSCF for the originating or terminating user
> sequentially towards app servers. Also app servers can send new
> requests, like for example I explained in a previous topic related to
> Application Server as B2BUA.
