Dear all,

Please perform an svn update on the ser_ims folder or simply check it 
out again.

If you used an svn update: make distclean should clean all the generated 
code.

Please use the command: make to check if the root executable ser is 
compiling. The output should start with:
aon at worf:/opt/OpenIMSCore/trunk$ make
bison -d -b cfg cfg.y
cfg.y: conflicts: 1 shift/reduce
flex cfg.lex

If you have errors here, than please check that you have installed bison 
and flex as mentioned in the prerequisites.

Then you can try to compile the modules using the command: make all.

I have also checked out the code separately and it compiles (including 
Client_Rf module).

Best regards,
Ancuta Corici
