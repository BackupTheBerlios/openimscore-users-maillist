Yes that's true.
When we did the last try about one year ago we recognized that there are some inconsistencies on Rx interface between P-CSCF and PCRF.
There was an evolution on P-CSCF but PCRF did not evolve in parallel.

I remember the following problem I discovered: The AA Request sent by P-CSCF in case of a registration (to enable a bearer for signaling traffic) does not contain a Media-Component-Number AVP.
This AVP is required and has to be set to value 0 in case of AF signaling. The missing AVP causes an exception in PCRF implementation of UCT.

So actually there is no running configuration.
