> Earlier this week  I had a problem with the Diameter peer. The hss and
> i,s-cscf are not able to talk to each other. I have checked my xml files
> and everything is file there. I have captured the packets using wireshark
> and attached the file. it seems that the hss are sending to port # 33462
> and the i,s-cscf are sending to port#33462,46157 I don't think thats
> normal,although all these ports beside the hss and i,c-cscf are occupied
> by the same process. can you please send me a feedback

Yes that's normal. As origin ports always "ephemeral" ports are used and the
responses have to be returned to theses ports. The destination ports are
3868, 3869 and 3870 as configured in the xml files. That's o.k.

But the messages are all empty (LEN=0) because these only show setup of TCP
connections.  You should analyse the setup of TCP connection and compare it
with your trace.
