I'm wondering why do you want to use SEAGULL? Do you have special reason 
for using SEAGUL? How about to try "IMS Bench SIPp" - 
http://sipp.sourceforge.net/ims_bench_sipp/reference.html

This is a performance testing tool based on SIPp and is conform to IMS 
Performance Benchmark specification, ETSI TS 186 008.

I used this tool two years ago for my thesis, testing OpenIMSCore. It is 
only SIP testing tool, but sending SIP messages to OpenIMSCore invokes 
also DIAMETER messages to the HSS.
I had to create bash scripts to "import" IMS Bench SIPp's users to HSS 
database, but I guess such scripts are generally available now.
