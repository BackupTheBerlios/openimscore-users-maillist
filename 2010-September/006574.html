Hi all,

Section 5.3.2.1 of 3GPP spec 24.229 says that3GPP TS 24.229

Upon successful user location query, when the response contains the URI of
the assigned S-CSCF, the I-CSCF shall:

 2) store the value of the "icid-value" header field parameter received in
the P-Charging-Vector header field and retain the "icid-value" header field
parameter in the P-Charging-Vector header field. If no "icid-value" header
field parameter was found, then create a new, globally unique value for the
"icid-value" header field parameter and insert it into the P-Charging-Vector
header field. The I-CSCF shall add a type 3 "orig-ioi" header field
parameter before the received "orig-ioi" header field parameter. The I-CSCF
shall set the type 3 "orig-ioi" header field parameter to a value that
identifies the sending network of the request. The I-CSCF shall not include
the type 3 "term-ioi" header field parameter;

The type 3 orig-ioi parameter is added when the communication takes place
between CSCF and AS. Why we are adding type-3 orig ioi in this case.

Also, when the request is forwarded to AS, we don't add any orig-ioi
parameter.
