Hi

 

We have observed one issue with OpenIMS Fokus HSS when our S-CSCF
interacting with it over Diameter Cx interface. When S-CSCF receives any
SIP Request with Authorization Header field containing
integrity-protected equals to "yes" or "tlsyes" or "auth-done", our
S-CSCF skips the MAR/MAA procedures and directly does SAR/SAA to update
the assigned scscf server name at HSS.. But looks like HSS is not
updating the assigned S-CSCF name on receiving the diameter SAR request.
Is this a known issue in OpenIMS HSS? Any work around or patches to
overcome this?

 

As a work around, our S-CSCF sends a dummy MAR request with authscheme
as "unknown" and Number of Auth Items as ZERO. But this MAR request
fetched us a MAA response with "DIAMETER_UNABLE_TO_COMPLY" as the result
code.

 

Any suggestions to resolve this issue..?

Srikanth
