Hi all,

IMS clients with IPSec support will be difficult to find for some
time. For non-IPSec clients, 3GPP defines early IMS security mechanism
in TS 33.978, which uses GPRS IP addressing information in the HSS
(obtained from GGSN) to authenticate the IMS user.
Support for this mechanism in the OpenIMSCore would mean some thing like:
- changing Cx to use the AVPs to convey the IP addressing info (we
could have a parameter to enable early IMS security, disabled by
default). This means support in I-CSCF, S-CSCF and HSS
- adding a script function in the S-CSCF to compare Cx-received IP
with the IP address in the Via of the message, asserted by P-CSCF
according to TS 33.978
- add IP addressing verifications in P-CSCF according to TS 33.978
- having the S-CSCF derive a Private User Identity from the Temporary
Public User Identity
- as there is no open GGSN (or is there?) for use with the
OpenIMSCore, could we have the IP addresses provisioned in the HSS?

What do you think? Is there enough interest to deserve a feature request?

JF
