Hi Joy,

For security issues, the I-CSCF is configured by default to accept only
INVITE, SUBSCRIBE or MESSAGE as initial requests. Normally, your ACK,
BYE and any other subsequent requests for dialogs should contain a Route
header when getting to the I-CSCF and be loose_routed before reaching
that condition. It seems that yours does not. Could you please check
that and if still in trouble post back a network dump?

-Dragos
