Hi Sudhakar,

Could you please provide to us the standard that this particular HSS
implements related to MD5?

The current MD5 authentication scheme approach is so due to the minimal
integration required - as it is now, there is practically no big
difference between AKAv1-MD5, AKAv2-MD5 and MD5. No standard was
followed, it's so just because it made sense.

We have also seen though other implementations that send the HA1. There
is a minimalistic advantage that the password is not transmitted in
clear. But anyway the Diameter connections should use IPSec.

So we are still looking for the standard that they followed in doing so.
We would like to converge, of course, but we don't want to just write
code and do things like that just because other people did differently.

-Dragos
