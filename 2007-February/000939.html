I am not sure if it is the same problem that you sent
before. As I had indicated in my earlier response, my
experience with SPhone was that it did not include
expires header. So S-CSCF does a SAR but tells HSS to
deregister the user as expires value is not present
(you log below shows 0 bindings). 
If you look at the SJPhone log it will say that
expires value is not returned by the server and a
default value is assumed. You need to change code in
scscf to address this.
