Hi Josef,

1. Do you actually have a presence server? By default the core is
provisioned to redirect SUBSCRIBE and PUBLISH request towards an AS for
presence. That filter is there for exemplification. If you don't like it
and want to do P2P presence, remove it. This is why you get a timeout.

2. I trace from the box running the core would've been more useful. The
trace that you have sent show only what happened with the clients, but
not what happened in the core and where the INVITE failed.

3. Why are you still using the SIP2IMS Gateway when we have MD5 and your
client also seems to support Service Routes?

-Dragos
