SJPhone does not include expires header in the
REGISTER request. So even though the authentication
succeeds, cscf treats it as a deregistration as
"expires"  header is missing. Since SJPhone receives
200 Ok it thinks registration succeeded and uses a
default expiry period. 
I have added the following code in
modules/scscf/registrar.c
to address this
        expires = cscf_get_expires(msg);
        if ( expires <0)
                expires = 3600;

- Sudhakar.
