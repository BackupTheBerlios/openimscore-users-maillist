Ankush, it seems to me that you did not read the relevant FAQ entry:
http://www.openimscore.org/docs/faq.html#sip2ims . All your questions
are answered there:


      SIP-to-IMS Gateway

    * *How do I provision a new user?*
      Use serweb or another tool to create a user in the credentials
      table. Alternatively, you can manually add a similar entry to
      those of Alice/Bob. HA1 is irrelevant and can be left blank. HA1b
      can be computed using e.g.:

      ser_ims/utils/gen_ha1/gen_ha1 user realm password

      Don't forget: The username must be the same as the private AND the
      public identity of the user, as SIP UEs do not usually make a
      distinction between them. The password must be the same as the
      secret key provisioned in the HSS. It seems that many users
      confuse the private identity, so here is a correct example:

      ser_ims/utils/gen_ha1/gen_ha1 alice at open-ims.test open-ims.test alice


-Dragos
