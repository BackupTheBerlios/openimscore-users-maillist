so... I tried a pkill ser  :-[ ... That obviously did not went well and
for sure this month I am not going to beat my uptime record with my
laptop :'(

then I RTFM and figured it out... But couldn't get what you got... Maybe
because I have a Pentium-M CPU?

In any case, there is a script ser_ims/cfg/killser that should do that.
It greps for the parameter and kills those processes. It would be
smaller if I knew about pgrep, but anyway does the job and does it
safely - it only kills ser processes.

call it like killser pcscf or killser cscf if you want all cscfs killed.
So far I did not had any trouble with that even on machines with 8 CPUs...

There is also a stopser variant which would be better to use if you also
use the persistency branch, so that the state gets dumped.
