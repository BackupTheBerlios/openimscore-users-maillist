Hi

I think you forgot to attach your opensips.cfg - It would help if you
included that wireshark trace as well, so we could see what the
presence server is doing with those messages (i.e. is it trying to
send them on somewhere else etc)

Do you have something like alias="open-ims.test" in your opensips.cfg
and SIP_DOMAIN=open-ims.test in your opensipsctlrc ?

Have you tried running opensips in debug mode ? This can be done by
setting the following options in opensips.cfg for example:

debug=6
fork=no
log_stderror=yes

And then running /etc/init.d/opensips debug

Let us know how it goes.
