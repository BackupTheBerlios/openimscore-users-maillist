Hi all,

after wasting a lot of time, I found out that the glibc6 in 
Ubuntu/Debian have a special desktop-user oriented patch for libnss-dns. 
So what happens is that there is no DNS query for AAAA (IPv6 resolve) 
sent from any components, in case you don't have any IPv6 address 
assigned, better than link-local, so the auto-detection mechanisms from 
the Open IMS Core do not work and an IPv6 setup on local-loopback only 
is not functional.

So if you have correct AAAA entries in your DNS but it still doesn't 
work and you only see type A DNS queries, make sure you provision an 
IPv6 address on your system. Also check the /etc/nsswitch.conf.

The name of the patch, in case you want to google for it, is 
local-ipv6-sanity.diff . This is not standard behavior, but it hides an 
issue on the transition to IPv6, recognized also in RFCs, with different 
DNS servers.
