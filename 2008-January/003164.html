Hi Trinath,

The problem would be that you have registered the following address:
Contact: <sip:amol at 10.50.41.196>;expires=3600

If there is no port, the default asumed one is 5060.

Then you are sending the INVITE below with a via of 10.50.41.196:4061.
This does not match the registered contact for the originating user
(sip:amol at open-ims.test) and as such it is rejected. Before using any
contact address for origination you need to authorize it through
registration. With the registration in the trace, by looking at the
trace, only the associated-uri's in the 200 OK can be used with only the
Contact addresses present there. Anything else could be an impersonation
attack.
