Hi Vladimir,

Hm... I should try it too :). Sorry, but we were using other commercial
tools for this or own-brew-load-generators so it won't be quite fair to
compare results in different situations.

Well, I can tell you that the 200 SAPS is quite real and with more
tweaking you could get even more. Yet there are some things that you
might have missed - the report stating the 200 should include all the
details about the tests, like the fact that a light-weight HSS was used
(C version, only implements Cx, very fast, similar to the CSCFs) and
improved CSCF configuration files as well (no logging, proper
dimensioning of the hash tables, large number of subscribers, etc). I am
not sure of the source of the report and might not be able to share many
of those with you because of NDAs, so please ask directly there for the
complete one. As the IMS Benchmarking standard that we wrote says (TS
186.008), the report should contain all the information that would make
the test duplicable.

On pure individual scenario, I have reached around 4-5 thousands of
routed SIP transactions per seconds, 5-7 hundreds of call
set-up/tear-down and 1-2 hundreds of registration - but this is
individual scenarios only, not a mix.

Cheers,
-Dragos
