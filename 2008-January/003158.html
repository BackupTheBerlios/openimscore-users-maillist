It's normal. By default we do not daemonize the CSCFs because the
project is all about experimentation and most of our users like to see
what is going on. The periodic logs tell you some of the current state
of the CSCFs.

If you would like to daemonize, see the ser parameters and change the
.sh file as you would like.
