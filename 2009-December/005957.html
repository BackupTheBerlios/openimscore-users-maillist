Hi Veena,

> The response from the ep2 to INVITE includes RECORD-ROUTE header field 
> with originating pcscf and scscf and terminating scscf uris.
> Terminating pcscf uri is not included.

After a quick analysis I found that the problem comes from the fact, that the terminating INVITE request is not recognized as an initial request but as a subsequent request. The function "P_is_in_dialog" checks based on host, port and transport if a dialog exists already. As the same host, port and transport has already been used for the originating INVITE (and therefore exists already) the P-CSCF concludes that the request is a subsequent one.
This leads to the problem that a Record-Route is not included anymore.

So I propose that you should use a more realistic scenario for your tests.
