Hi All,

When UA sends Require header with multiple tag like, secagree, 
precondition etc in PRACK requests.
----------------
Require: secagree, precondition\r\n
----------------
OpenIMSCore (r697) forwards the PRACK Requests without terminating 
Require header with '\r\n'
when it removes the 'secagree' option tag in the list.
---------------
Require: precondition\n
---------------

Please see the PRACK request in attached TLS call capture. this behavior 
is also observed for INVITE request also.
Are we missing anything? please clarify.

Since Packets are encrypted in TLS,  To decode the packets in wireshark 
use attached 'pcscf_private_key.pem' file
in wireshark->Edit->Preferences->Protocols->SSL.
RSA Keys list===>172.16.8.107,4061,sip,/file_path/pcscf_private_key.pem
