Update:

It seems that ACKs are never routed to AS. The function 'ISC_match_filter'
that is called
in scscf.cfg is never called for ACK so they never get sent to AS.

I've changed the S-CSCF file to include a filter for ACK coming in from
user.
I've also removed an 'if' loop that blocks ACKs being filtered in
'ser_ims/modules/isc/sip.c', line 114.
