Sudhakar Patil wrote:
> Hi, 
> In the scscf.cfg, currently a check is not made if the
> terminating public identity is barred. According to
> 24.229 section 5.4.3.3 S-CSCF is supposed to check for
> barring of terminating user and send 404 if the public
> identity is barred. Without this check terminating
> calls will go through. The following lines of code
> might need to be added to scscf.cfg to solve this. 
>
> route[Term]
> {
>         log(1,">>       Term\n");
>
>         if (S_terminating_barred()){
>                 sl_send_reply("404","Not Found -
> Terminating user not found");
>                 exit;
>         }
>
