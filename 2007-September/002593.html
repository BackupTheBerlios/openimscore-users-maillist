Hello Melbourne,

The OSIMS does not have any RTP stream capabilities. The only thing that
it does is that it can manipulate the SDP in SIP messages so that the
UEs would talk through RTPProxy (however, the RTP proxy just proxies the
RTP packets without any decoding).

Could you use the same encoder to also decode? It looks like G.729 is
patented by Sipro...

-Dragos
