In my experience "Awaiting Proxy login information" means that X-Lite is not
even trying to register; it's waiting for you to fully configure a SIP
account. As I wrote in an earlier response, I haven't used the Linux version
of X-Lite for a long time but on Windows, I use the following settings with
the Open IMS. The system where you run your X-Lite must of course be able to
resolve pcscf.open-ims.test (check with ping or host pcscf.open-ims.test).

Enabled: Yes   {Don't forget this one, otherwise all other data is
discarded!)
Display Name: <whatever you like>
Username: alice
Authentication username: alice at open-ims.test
Password: ****
Domain/Realm: open-ims.test
SIP Proxy: <empty>
Outbound Proxy: pcscf.open-ims.test:4060
Use Outbound Proxy: Always

Hope this helps,
-David

Hi all,

"awaiting Proxy login information" - which client are you using Amol?

I've seen in a mailing list lately that UE config was not correct when the
latter returns this error.

Check that outbound proxy == pcscf.open-ims.test (4060)
and registrar == open-ims.test

In your DNS you should perhaps declare the hostnames and respective ip addrs
for each server
