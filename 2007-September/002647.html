Hello Dragos, 

I am a bit confused. PacketCable 2.0 spec
PKT-SP-29.228-I01-060914 (Table 6.3.2) specifies that
S-CSCF shall set SIP-Authentication-Scheme in MAR to
"Unknown" if the S-CSCF supports multiple
authentication schemes. The spec does not indicate the
use of "Digest" in the MAR request. However, the spec
does specify that in the MAA (Table 6.3.6)
SIP-Authentication-Scheme would be set to "Digest". So
it is "Unknown" in the MAR and "Digest" in the MAA. 

Based on your response it looks like we can achieve
this by using "HSS-Selected" option in the scscf.cfg.
Am I right? Still, I think if someone does set the
default algorithm to "CableLabs-Digest" s-cscf would
not set the right MAR message to HSS according to the
spec.

Regards,
Sudhakar.
 

Hello Sudhakar,

No, it's not a bug. You requested "CableLabs-Digest"
and that is the
identifier for the Digest authentication from
CableLabs. If you want to
get a Unknown identifier, then please select
HSS-Selected.

Also please be aware that the NASS-Bundled
authentication is still not
in a trunk, but in a branch. Will be merged soon.

-Dragos
