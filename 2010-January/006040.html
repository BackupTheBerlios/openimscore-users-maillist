> Is there anything necessary to configure an OpenIMS environment to be able
> to interoperate with another IMS realm?

You have to 
- setup the two domains in DNS pointing to the I-CSCFs 
- enable roaming for the user
Then it should work.

> Currently, when I try to, say send a SIP MESSAGE, from my IMS dev realm to
> my OpenIMS test realm the test server drops the message because it
> interprets it as an originating request and (obviously) the sender is not
> registered with this IMS realm (since it is only registered in the dev
> realm).

The S-CSCF of "IMS dev realm" the originating network should automatically
forward the request to the I-CSCF of "OpenIMS test realm" if DNS is setup
correctly.
