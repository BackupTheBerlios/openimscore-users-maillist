Hi Rivaldo,

If you want to use the 3G interface on the mobile handset, you will
need 3G infrastructure (BS, GGSN, AAA etc.). If you have access to
this infrastructure, then you can run the test in the lab. If you
don't have access to this infrastructure, you can make your OpenIMS
core listen on a public IP address and use your 3G service provider's
infrastructure to route SIP messages between IMS client on the mobile
handset and the OpenIMS core. However, with this approach, you may not
be able to test the PCRF and QoS functionality since you are using
your own IMS core which is not allowed to talk to the service
provider's PCRF, GGSN etc. If you don't have access to a public IP
address, and if you have a dual mode handset, you can just use the
wifi interface on the handset to route packets to the OpenIMS core in
the local network.
