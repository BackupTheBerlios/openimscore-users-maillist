Hi Francesco,

> I have attached the trace and console log files of pcscf node. At the
> end of console log I noticed a parse error. 

The logfile (and as you already recognized) shows clearly the problem: 
At registration the client (UCT IMS Client) includes a +sip.instance
feature-tag 
   +sip.instance="<urn:uuid:679dbcb6-04e8-11df-9e0b-f119edf4afb6>"

This feature-tag is obviously not recognized by S-CSCF and as a result it
returns a NOTIFY-request for the reg-event with an XML including a hint:
	<unknown-param
name="+sip.instance"><urn:uuid:679dbcb6-04e8-11df-9e0b-f119edf4afb6></unknow
n-param>

This XML-hint cannot be processed at P-CSCF and therefore it rejects the
NOTIFY with:
	500 Error encountered while processing notification

So the next question is if a +sip.instance feature-tag can be processed by
S-CSCF. I think anyone of the core-team should easily answer that question.
