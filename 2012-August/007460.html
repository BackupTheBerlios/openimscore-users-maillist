> First - when I want to use binary sms, I have to specify SMS Contact
Center in
> the Boghe configuration. Then, Boghe sends SIP Mesaages with address of
> this contact center in the "To" field.
I never checked that feature of Boghe, but I expect that this address is
also used as request URI.

> In its turn, OpenIMSCore replies that "HSS User Unknown".
Thats clear, as the "SMS Contact Center" is not a user. It has to be an
application server.

> What is mean by  "Contact Center"? SMS gateway (sms module)? 
This obviously is the IP-SM-GW, an application server from IMS perspective.

> And how it can be registered  in the network?
The IP-SM-GW address has to be handled as a PSI (Public Service Identity).
There are several methods to provision a PSI.

> Second - Boghe sends sms in binary format only when destination sip

This seems reasonable for me, without digging in the standards.

> Also, it's not very clear for me how MESSAGEs can be routed to the sms
> module.
