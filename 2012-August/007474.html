Hi,

We have compiled P-CSCF to enable Rx interface using code "http://svn.berlios.de/svnroot/repos/openimscore/ser_ims/branches/08_cdp_sessions".
However, we observed issue in Registering user with this new P-CSCF when Authentication algorithm is used as AKAv1-MD5.

We observed when P-CSCF received 401 response from I-CSCF with WWW-Authenticate header, P-CSCF was incorrectly forming this header such that there is no comma "," between Algorithm and qop parameter. As a result, client sends Authorization header with md5 algorithm.

Please note that this issue is not observed if P-CSCF is used without PCRF code. For analysis, I am attaching ethereal logs of this scenario also.
