> I wonder if pstn support only works if there is a tel uri as another
public
> impu? 

Yes that's true. You must always provide a second IMPU (a TEL URI) in an
implicit registration set,. Then the originating S-CSCF automatically adds
the TEL-URI to the P-Asserted-Identity header field.
Only this mechanism allows the termination in PSTN with a proper CLI
inserted.
