Hi,

Could anybody explain me what is the required server/client configuration
for sending sms in binary format?
At the moment I have OpenIMSCore server with enabled sms module and two IMS
clients (Boghe) on the different PCs.

First - when I want to use binary sms, I have to specify SMS Contact Center
in the Boghe configuration. Then, Boghe sends SIP Mesaages with address of
this contact center in the "To" field.
In its turn, OpenIMSCore replies that "HSS User Unknown". What is mean by
"Contact Center"? SMS gateway (sms module)? And how it can be registered in
the network?
Second - Boghe sends sms in binary format only when destination sip address
in telephone number format is used. e.g. contact address is
sip:+3809876543 at open-ims.test
Is it common rule for IMS?

Also, it's not very clear for me how MESSAGEs can be routed to the sms
module.
At the moment I have such configuration in the scscf.cfg:

==============================
modparam("sms","modems","Nokia [d=/dev/ttyACM0;b=9600]")
modparam("sms","networks","D1[m=10]")
modparam("sms","links","Nokia[D1]")
modparam("sms","domain","open-ims.test")
modparam("sms","max_sms_parts",3)
modparam("sms","use_contact",1)

# main routing logic

route{
        if(method=="MESSAGE"){
                sms_send_msg_to_net("D1");
        }
...
}

==============================

As a hardware GSM gateway my cell phone is using.
But, as I see, nothing happens when s-cscf processes SIP MESSAGEs. I don't
see that these messages are forwarded to the sms module, or smth like that.

Maybe someone have already faced with OpenIMSCore configuration for
processing binary sms, so could you share needed for it configuration,
please?

I would be very appreciated for any help.
Thank you in advance.
