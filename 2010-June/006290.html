Hi Kapila,
 
> It is mentioned in some specs that the charging can be feature dependent.
> For example, for a call initiated by a particular subscriber we can use
> onine charging and for MESSAGE we can use offline charging. I am unable to
> find any field in the user data in which this information is used. At the
> time of registration, we get both the CCF and ECF addresses, then how can
> we decide whether to do an online or offline billing of this subscriber.

You are mentioning two different aspects of charging control:
- service specific
- subscriber specific

The service specific handling can be controlled via selecting which node
should generate a CDR for which event. This should be operator configurable.
The subscriber specific handling can be controlled vie the
Charging-Information provided by the HSS.
