Hello Antonio,

I think the error message "Name or service not known" is generated
because the host hss.open-ims.test was not successfully resolved.

To test this, use ping hss.open-ims.test. If the IP you wanted is shown,
than:
1) either the HSS is not running
2) either the interface between the HSS and I-CSCF/S-CSCF is broken

In the case that you are using another IP/DNS configuration that the
default one, for the HSS you can check the configuration files:
FHoSS/deploy/DiameterPeerHSS.xml and
FHoSS/deploy/hss.properties

For the I-CSCF and S-CSCF diameter interface you can check the
configuration files:
ser_ims/cfg/icscf.xml and ser_ims/cfg/scscf.xml

(Acceptor port is the XML tag that tells you where the HSS/I-CSCF/S-CSCF
is expecting diameter messages)
