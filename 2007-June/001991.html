I had similar problem once, when there was an error in DNS configuration and machine with X-Lite were unable to found a DNS entry for IMS domain. ACK is being sent in a transaction, where X-Lite should resolve (usually for the first time during the call setup) the names specified in route set.

Try capturing not only SIP, but also DNS and make sure that all necessary DNS names are resolved.
