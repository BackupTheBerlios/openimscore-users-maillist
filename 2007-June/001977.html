Hi,
we are testing a resynchronization procedure and our UE generates an 
auts token auts="VFxOD2Jj0NJlgRCHUMU=" the rand and auts parameters in 
hexa are as follows RAND=df1df3dd6df0670ed661c3f1e6043e21 
AUTS=545C4E0F6263D0D26581108750C5. When an scscf recieves auts token it 
generates an MAR diameter message with the SIP-Authorization
but in this field there are 3 parameters as follows RAND | AUTN | AUTS. 
In 3GPP TS 29.228 in table 6.3.3 you can see that SIP-Authorization 
field should contain only RAND and AUTS. So in my opinion s-cscf should 
send only RAND and AUTS in the MAR message. Our HSS expects only these 2 
parameters.
