I'm having an issue with the request routing on the PCSCF.  I have an application server which implements a B2B agent for VOIP calls.   All requests get routed through my server. However, after the initial call setup is completed, if the terminating client sends a BYE request the PCSCF sends the request directly to the originating client. The Route header in the BYE request seems to be correct. Also if the originating client sends the BYE, the request gets routed correctly to my Application server.

I've attached a packet trace to this message.  The originating client is 10.247.161.189, the terminating client is 10.247.161.213, The IMS core is 10.247.160.50 and the app server is 10.247.160.249

I'm also seeing the same behavior when I send subsequent requests from the terminating side. For instance I'm trying to place a call on hold from the terminating side. The invite from the terminating side is sent to the PCSCF but is not forwarded to the SCSCF.

I'd appreciate any suggestions/help, thanks
Paul
