> I would still like to know the root cause which I believe lies with OpenIMSCore...

I don't think so. I just checked your issue.
Unfortunately the pcscf.log does not tell so much, because you did not increase debug level.

But I guess the situation is as follows:
The REGISTER request has the following Via:
	Via: SIP/2.0/UDP 172.16.175.74:5070;rport;branch=z9hG4bKPj7F-Xg1UfbziernQ1LUQlh3pAUFwcefRf

The Via in your SUBSCRIBE request is
	Via: SIP/2.0/UDP 150.254.41.175:5070;rport;branch=z9hG4bKPjdLLU9QWFcTVaSnQIAClzudldnPPGHtnV

This is a different sent-by address. 
The P-CSCF has to check if requests come from valid (registered) subscribers. If it is a different address the P-CSCF decides on a different subscriber who has to register first.

Why does the SUBSCRIBE contain another sent-by address?
