I am integrating SIP Application server and a SIP Client with OpenIMS in a
sandbox server. The application server is acting as a B2BUA

I configured succesfully filter criterias for the Application server and
the mesasges seems to route succesfully

If A calls B through the Application server and A cancels the call before B
accepts  the application server will generate a CANCEL that will be
rejected by PCSCF with following message:
        sl_send_reply("403","Forbidden - You must register first with a
S-CSCF");

message.  But even when I add manually that recourd-routes as routes of the
CANCEL it keeps failing with the same 403. Which could be origin of the
