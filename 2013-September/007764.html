Finally could fix the issue

The problem was that  my application server was receiving two record-routes
in the ringing response but not using them properly
When originating the CANCEL Message it was inserting the routes in the same
order they were received n the RINGING message instead of using the reverse
order so the first route of the CANCEL from Application server to B  was
PCSCF instead of SCSCF.despite the message was going first to SCSCF which I
guess is wrong.
