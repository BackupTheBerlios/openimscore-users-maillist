Hello!

I try to run OpenIMSCore with Monster client above IPv6 and encounter a
problem, which I can't solve. I create two users (test2 and test3) in the
HSS web console and I'd like them to communicate with each other using
Monster, but I always get an error: "Error while sending message: 403
Forbidden - You must register first with a S-CSCF". The strangest thing is
that I try this whole procedure using Monster above IPv4 and Linphone above
IPv6 and in these cases everything worked without any problem.

I investigated the SIP messages with Wireshark and found that the
Register->401 Unauthorized->Register->200 OK message sequence is all right,
but after this, when the client sends Publish and Subscribe requests (and
Invite when I try to make an audio call), the responses are 403 Forbidden -
You must register first with a S-CSCF.

According to the HSS web console, the users are registered and they are
assigned to an S-CSCF.

I use OpenIMSCore revision 1007 and Monster 0.9.13, but older Monster
version and OpenIMSCore revisions also result the same error. I searched the
archives, but I can't find any solution. Do you have any idea what is the
problem? I'm looking forward to your answer(s).
