Hello Mustafa!

> i am trying to send SIP OPTIONS message using SIPp to Open IMS Core;
> but i get this error:
> 
> SIP/2.0 406 Initial Request Method not allowed at the I-CSCF;
> is this mean that I-CSCF does not support SIP OPTIONS message;
> and how i can use sipp to test pcscf compatibilities Or any SIP proxy?.

This is caused by I-CSCF which actually is restrictive on handling SIP
requests with the statement:

	if (method=="INVITE" || method=="SUBSCRIBE" || method=="MESSAGE" ||
	    method=="INFO" || method=="PUBLISH" || method=="CANCEL"){
		route(Initial_Request);
		break;

As a quick shot one could propose to enhance the stemement with "OPTIONS",
but that might not solve your problem. This depends on the target of the
OPTIONS request. If OPTIONS is used to query PUIDS then it should work. If
OPTIONS is used to query a server of the IMS core it will not work, because
the "route(Initial_Request);" leads to a diameter LIR.

So, what do you do with OPTIONS?
