Hello Mustafa,

> I would ask you about 100 trying message in OpenIMS;
> what is the status that OpenIMS core nodes generate the provisional
> response (100 trying); how i can generate 100 trying response for every
> SIP request like (registration;Invite;...).

You cannot do that. Automatic generation of a 100 Trying response is only
defined for INVITE request in RFC 3261, and this is done by OpenIMSCore.

> When UA receive 100trying; UA will adapt its retransmition timer T1 to the
> calculated value from timestamp;
> then UA adapt T2 Timer value; is This true?

I am not sure if I have fully understood your question.
The behaviour of a UAC receiving the timestamp value is undefined in RFC3261
§20.38. In addition the timestamp mechanism allows only to estimate the RTT
on one hop. There is no mechanism defined to forward a timestamp value at a
proxy.
