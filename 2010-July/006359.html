Hi all,

I try an AS (B2BUA) Integration with OpenIMSCore. 
Unfortunately the used AS add some proprietary SDP media attribute lines
to the SIP Status 200 ok message which make the IMS-to-PSTN and
PSTN-to-IMS calls fail.

I am looking for a workaround to remove this SDP media attribute lines
again.

Right now I am using subst() function from module textops to remove the
SDP media attribute lines in S-CSCF.

This works fine for the IMS-to-PSTN call.
I added the subst() call before the S_update_dialog("orig") call in
section onreply_route[ISC_Orig_reply] of scscf.cfg file.
The SDP media attribute lines are removed and the SIP content length
header gets the new length value.

I tried the same for the PSTN-to-IMS call.
Therefore I added the subst() call before the S_update_dialog("term")
call in section onreply_route[ISC_Term_reply] of scscf.cfg file.
The SDP media attribute lines are removed but the SIP content length
header still has the old length value.

Does anybody know what could be wrong?

Does anybody know a better workaround for such a problem? 
  
Thanks in advance,
Werner
