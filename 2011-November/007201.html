Hi,

In my traces I find the following. An AARequest that has the following AVPs:

Session-Timeout AVP with value of 60.
Authorization-Lifetime AVP with value of 296.

RFC3588 states:

> 8.9. Authorization-Lifetime AVP
> 
> … If both this AVP and the Session-Timeout AVP are present in a message, the value of the latter MUST NOT be smaller than the Authorization-Lifetime AVP.

And:

> 8.13.  Session-Timeout AVP
> 
> ...  When both the Session-Timeout and the Authorization-Lifetime AVPs are present in an answer message, the former MUST be equal to or greater than the value  of the latter.

So, the value of the Session-Timeout AVP should be higher or at least equal to the Authorization-Lifetime AVP.

It seems that the current Rx implementation in OpenIMS core is not correct on this point.
