Hi Frederic,

Again, a trace would be helpful.

The PRACK is a subsequent request. Does it contain the proper Route
headers, as indicated in the Record-Route headers in the 183? If it
does, then it should contain the whole path that the INVITE went through
and routing should be OK.

As a general security rule - if you don't want the simplest
impersonating, spam, etc attacks, your client should ONLY speak with the
P-CSCF. So it's a good practice to also put it as a strict outbound proxy.
