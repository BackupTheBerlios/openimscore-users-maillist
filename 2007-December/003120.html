Hi Vladimir,

I believe the FOKUS result you heard about was with a SUT comprising a C-written HSS instead of the Java FHoSS included in the open source project.

Java does not feel like the first choice for implementing an HSS solution where performance matters (not that it is necessarily a bad choice either, just a matter of compromise). In the open source project, ease of development and extendibility were probably favoured instead of performance. Also, FHoSS probably was never architected or tuned for performance.

I unfortunately cannot confirm the result you achieved with FHoSS in the SUT because we've never really tested that combination either.

Regards,
-David
