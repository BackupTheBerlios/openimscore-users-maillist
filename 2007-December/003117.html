Hi list!

To answer to the couple of questions regarding policy support in OpenIMSCore.

There is a branch 08_cdp_sessions with a version of the P-CSCF that supports the Gq or Rx interfaces.

There is a basic readme in the documentation doc/README.PCC 
(http://svn.berlios.de/svnroot/repos/openimscore/ser_ims/branches/08_cdp_sessions/doc/README.PCC)
I also included wireshark dictionaries and all the files needed to run a test with Seagull as your PDF/PCRF 

There is no PDF or PCRF offered Open Source with the OpenIMSCore. 
If you urgently need a PDF/PCRF there is one Open Source from the University of Capetown,
based in the Java Diameter Peer provided with OpenIMSCore.
 http://uctimsclient.berlios.de/policy_control_howto.html
if you want to know about other posibilites we can talk!

The interfaces are still under development and therefore a bit buggy but it works quite fine.


If you want to stop a call consider on some circunstances consider using the functions under the release_call.c/release_call.h

There are some known bugs to be solved there, but as this is also in the trunk i am running some tests on the patch before updating it (do for January 2008)

If you have an interest in this area , just drop me an email, I reply! and we can see what we can do together
(also if you are looking for Charging, RACS interoperability (Gq')...)


I hope this helps!
