>      Please help and tell me why? Please see trace in attachement.

You client uses different ports for each transaction:
- 1909 for 1st REGISTER
- 1910 for 2nd REGISTER
- 1911 for SUBSCRIBE
- 1912 for MESSAGE 

This is a very strange behaviour.
The P-CSCF has to check if requests come only from well known (registered)
clients. Therefore the SUBSCRIBE and the MESSAGE request are rejected by
P-CSCF with "403 Forbidden - Not Registered! You must register first with a
S-CSCF"
