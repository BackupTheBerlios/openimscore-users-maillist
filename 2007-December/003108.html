Hi Sabyasachi,

PSTN = Public Switched Telephony Network, or how we smart-NGN-asses call
it, Plain Stupid Telephony Network ;-). The default 127.0.0.1:9060 would
need to be replaced with the address of your gateway to PSTN.

so, you are trying to call a TEL URI. In this case the S-CSCF tries to
route towards the PSTN as it thinks that, since the ENUM query failed on
it, this could be the only way to handle it.

Solution: use enum and indicate that your domain is servicing that TEL
URI. Or don't use TEL URI....(or, as always, just hack the cfg file).

Also I see that you are using a very old revision (268?). I recommend
that you update to the latest one, which has a much improved ENUM and
TEL URI handling.


Cheers,
-Dragos
