> 1. Has anyone used the SIP Digest authentication method ? If you are 
> aware of the procedures used in SIP Digest authentication, can you 
> please point
me
> to any links or references?
> How different is it from AKA or the early IMS authentication methods?

The main difference is:
- With  " SIP Digest authentication" the client does not provide an Authorization header field in 1st REGISTER request (this is the usual way in a SIP registration)
- The missing username parameter in the Authorization header field is derived from the "From" header field. Therefore IMPI and IMPU have to be aligned accordingly.

To configure SIP Digest authentication for a user you have to set:
-  "Let HSS decide"  in authentication section of scscf.cfg:
    modparam("scscf","registration_default_algorithm","HSS-Selected")
- for the authentication method of the user in HSS: default =Digest-MD5

> 2. Can TLS be used only over SIP Digest or over any authentication 
> mechanisms? Because I see testcases mentioning "SIP Digest with TLS" 
> or "SIP Digest without TLS".
No, this only makes sense for the two mentioned methods. 
E.g. TLS with IMS AKA is not defined and does not make sense.
