Hi Badri,

> In case of NASS Bundled authentication (NBA), there is a optional
parameter
> (Authorization) that can be put in to the REGISTER request. Does anyone
> know the purpose of this optional parameter?

To identifiy the user. See also TS 33.203 Annex P.6.:
For NBA the inclusion of an Authorization header in a registration request
is optional. However, it is specified for NBA in TS 24.229 [8] (c.f., clause
5.3.1.2) how to derive a private user identity from a public user identity.
This derived private user identity is then used in Cx commands.

> Because, the authorization is going to be line-authentication anyway. So
> what will this optional parameter achieve in NASS.

... the identity of the user.

> If it does make sense in NASS -
> I have also read that the private identity should be put into the
Authorization
> header. ( is it into the username part of the header??? ) & Can the Realm
and
> Password be left blank in the Authorization header then?

Yes, it is the username parameter of the Authorization header field.
The realm parameter is required for routing the request to the "right"
network.
There is no "password" parameter.
