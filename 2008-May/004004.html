  We have 2 Application servers connected to the same Open IMS domain ( AS1 and AS2).
  We would like to send a SUBSCRIBE from AS1 to AS2 through the CSCF and then get a 200 OK answer back from AS2 to AS1 via CSCF.

  When we send the message from AS1 to AS2 through the S-CSCF we get '404 Not found - destination user not found on this S-CSCF'.
  (We set the To-field to AS2 but we set a Route header to point to S-CSCF)

  Note: When we send SUBSCRIBE from a user to the servers it works perfect. The message go througt to both of them depending on the To-header in the message.

  3. Is the  Profile Part Indicator of the IFC attached to the AS influencing whether it should be registered or unregistered?
