Hi Robin,

Unfortunately, the 3GPP specs do not indicate the possibility of using 
regular expressions in the Request-URI. I find this strange, but it is 
implemented like this right now.

I would propose extensions, as this is often quite bothering. Now we 
triggered instead on the To header, as for headers you can use regular 
expression, but this is not so good. Yet such an extensions should be 
backwards compatible, so we probably would need a flag in the SPT. This 
could be put maybe as an extension? Then the S-CSCF code changes in the 
isc are not a big deal as most of the stuff is already done.

I don't think that I can find time for development right now, but if you 
could help I will give you all my support.

Cheers,
-Dragos
