This (address of I-CSCF) makes more sense. See also the attached scenario.
As a first quick answer:
The Asterisk as an application server for IMS does not behave as it should
as an AS. This I have expected because Asterisk is not designed to act as an
AS for IMS. E.g.
- It does not use the Route header in F10 (continue the Route) provided
reviously by the S-CSCF (F8)
- It does not provide a P-Asserted Identity header.

The request (F10) sent by Asterisk is now handled as an incoming request
from a non IMS network addressed to john and obviously forwarded again to
Asterisk based on a terminating iFC. Then Asterisk classifies the request as
a looped request because F15 has the same Call-Id as F10.

The next questions now are: 
- What application should Asterisk offer in this scenario?
- Which iFCs are in place for jane and john and why?
